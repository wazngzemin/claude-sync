全局视角：这张图在讲什么

一句话：一个用户请求或一个传感器信号，从进入系统到最终执行或回复用户，经过了哪些模块、做了什么判断、数据怎么流转。

三方架构（6.22定版）：

层归属模块链路
云端字节服务器⑨拒识仲裁 → ⑬规则配置 → ⑮触发器引擎 → ⑯⑰Advisor → ⑲⑳Director → ㉒工具库
端侧AIBox字节①②用户输入 → ③④信号库 → ⑤⑥白名单 → ㉔端侧触发器 → ㉕感知器
CDC底软赛力斯固定闭环（起步安全/迎宾/充电大师等）→ 直读信号 → 直接执行

端云分界线就是图中间那条虚线——每条穿越它的橙色粗线都要走网络，弱网会卡。

四条铁律（6.22定版）
  1. 动态注册经⑮云端触发器中转
  2. 端结果 → ④信号库 → ㉑Context（不直推不跳级）
  3. 触发器只读④信号库（不碰原始信号源③）
  4. condition → ㉑Context供读取 / event → ⑮组装push给Advisor（6.22最核心）
一
端侧输入层：信号从哪来
① 用户语音query → ② ASR语音转文字

用户在车里说了一句话。①拿到原始音频，②在端侧做ASR转成文字。

① 用户语音 → L1 → ② ASR转文字 → L2(橙色跨端云) → ⑨ 拒识仲裁

③ 端信号源（6类）→ ④ 端侧信号库

除了语音，车上有6类传感器持续产生数据：

类别举例

3. 埋点App操作 / 点击 / 滑动
4. 电子围栏进出区域（[需10/11]）
5. 音频音量 / 切歌 / 循环（[需13]）

+ 赛力斯CDC事件接续导航 / 展车点击等

j. 详解——6.22最关键的变更

❌ 之前的理解（错误）

Advisor轮询Context拿状态变化 → 自己判断是不是"事件"

✅ 6.22纠正（正确）

⑮负责做"状态变化检测 + 事件组装"，然后push给Advisor。Advisor只管等push。

具体机制：

1
⑮持续监听⑫云端信号库的信号快照
2
检测到阈值穿越（last=F & this=T），如"档位从R变为P"
3
组装成语义化事件："倒车完成"
4
push给订阅了此事件的Advisor

⑮的四个输出方向：

连线方向说明
L17（粗线）→ ⑯ 静态Advisorpush事件
L18（粗线）→ ⑰ 动态Advisorpush事件
L21（虚线）→ ⑲ Director条件已触发通知
L14（橙色粗线）→ ㉔ 端侧触发器场景下发到端侧执行
佳杰原话："我不会去轮询你的Context。我注册了事件之后，就等待你的云端触发器给我发这个事件。"
铁律④诠释——佳杰经典拆解
佳杰："倒完车之后，如果车里有小朋友，就把儿童锁打开"
成分对应数据来源
"用户倒完车"事件 event等⑮ push
"车里有小朋友"条件 condition读㉑ Context
"把儿童锁打开"动作 actionAdvisor给出

6.22 scope收窄——仁杰判定标准

归CDC底软

  • 常驻 / 固定 / 标配 / key-value
  • 起步安全、上下车安全
  • 睡眠提醒、无感迎宾
  • 充电大师、哨兵模式

归端侧触发器

  • 有生命周期 / 动态 / 可运营 / 临时
  • 云端下发的定时/条件任务
  • 动态视觉感知事件
  • 通用视觉通道（兜底长尾）
仁杰原话："像起步安全，你放在触发器里面专门去配一下，反而还要让触发器去接底层信号、考虑稳定性和延时，不如直接在底软就把事做了。"
弱网执行

网络断了，㉔可独立执行：→ ㉘车控执行（弱网本地执行）→ ㉖TTS播报（弱网TTS）

例：车进隧道，云端已下发"到目的地500米提醒"。㉔本地定时器到时 → ㉖TTS "快到目的地啦" — 不需要云端。
八
CDC底软 · 固定闭环区（6.22大幅扩展）

以下功能全部归赛力斯CDC，不走触发器：

功能为什么归CDC
起步安全所有车标配，纯key-value判定
上下车安全固定规则，不需要动态配置
睡眠提醒系统设置button，常驻功能
无感迎宾系统feature，不是场景引擎
充电大师赛力斯自己的业务逻辑
哨兵模式赛力斯端侧闭环

数据流：直读④端侧信号库 → 直接执行㉘车控/㉖TTS，不绕触发器链路。

西钺原话："不要开发两个触发器，不管从开发工作还是部署资源上都是浪费。端侧触发器只在CDC，VLM在AI Box。"
九
6.22新增支撑模块
任务中心（端侧）—— 佳杰依赖

端侧需要一个持久化任务库：

需求说明
任务级 vs 感知级用户能看到的"帮我照看宝宝"=任务级，底层注册的"观测毯子位置"=感知级
enable/disable接口用户在任务中心UI上操作，端侧要真的关停对应任务
端云任务同步端侧任务库 ↔ ㉓云端任务中心双向同步
AIBox任务调度

优先级排序在佳杰的AIBox模块里做，不在触发器。always-on、VQA、其他视觉需求——优先级低的排队等。

通用视觉事件通道

兜底长尾的"万能通道"。当预设事件集不够用时，透传动态query给AIBox：

分类说明
舱内静态性别、年龄等长时间不变的
舱内动态动作、表情等快速变化的
舱外只用前摄
关键：prompt是增量追加，不是覆盖。否则加了新的观测项，原来的感知就没了。
