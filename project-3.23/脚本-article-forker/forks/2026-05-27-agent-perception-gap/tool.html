MODULE 3"优雅失败"诊断——说了但没做对
学习目标：识别Planner的"合理解释"通病——talk_content很流畅但action_list错了。这是最难发现的badcase类型。

图3：三种失败模式对比 — "优雅失败"是最危险的，因为你以为任务完成了

Prompt模板：排查"优雅失败"

以下是5条Planner输出日志，每条包含user_query、talk_content、action_list： {粘贴日志} 请逐条检查是否存在"优雅失败"——talk听起来合理但action不正确： 判断标准： 1. talk_content承诺做的事，action_list是否真的做了？ 2. 用户隐含的需求，action_list是否覆盖了？ 3. action_list为空时，是真不需要操作，还是Planner没能力理解？ 4. 工具参数是否精确（位置、时间、数量等关键参数）？ 对每条标记： ✅ 正确执行 ⚠️ 部分执行（漏了什么） ❌ 优雅失败（说了但没做对） 最后统计：优雅失败率 = ❌数量 / 总数量
场景：用户说"明天早上叫我起床"，Planner回复"好的，明天早上我会叫你起床的！"，action_list为空。这是什么？
正确。这是典型的优雅失败：talk让用户安心（"好的"），但action_list为空意味着没有任何实际操作（没有调用goal_list_update设定条件任务）。用户会以为明天会被叫醒——结果不会。
不对。Planner不会"记住"——它是无状态的。如果action_list为空，什么都不会发生。用户被流畅的talk_content欺骗了。这就是"优雅失败"。
