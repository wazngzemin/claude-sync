三运行时全链路演示：一家三口上车，T0→T9

设定：爸爸（主驾）、妈妈（副驾）、5岁娃（后排右）。时间从上电开始。

T0 00:00车辆上电，感知系统就位SLS→端侧触发器→AIBox

T4c分支三：没人理，3分钟后卡片消失

五坑清单

🔴 K1（阻塞）时效坑：儿童识别挂在5分钟低频任务上，"上车即推荐"最坏要等5分钟
方案：把条件①占座变更事件同时配置为oneshot插队识别的触发源——上车瞬间插队跑一次乘客识别，不等低频周期。依据：静态会丁彬确认的插队机制（"高优one shot会把当前alwayson抢一次"）。这需要在配置平台把"事件→oneshot识别→再判条件"配成链，和充电桩case（R→P→看充电桩）同构——两个case一起提，链式触发就是通用能力而不是特例。
🔴 K2（阻塞）事件×状态混合AND的判定语义没定义
条件①是事件（变更为，瞬时沿），条件②是状态（等于，持续），两者到达时间差最长5分钟。若15-4把AND实现成"同一时刻两个都在变更"→永不命中；若实现成"两个都是当前状态"→主驾一直坐着，中途儿童被抱上车也会触发（可能是对的？也可能不是产品预期）。PRD必须写清：AND组合里事件条件的"有效窗口"（如事件发生后N分钟内状态补齐算命中）。这是所有"上车+X"类场景（事件库里挂了30个场景）的共性问题，一次定义全家受益。
🔴 K3（阻塞）"拒绝后本次上电不再推荐"没有实现闭环
execute_policy只数次数不懂"拒绝"。需要：Director/Advisor拒绝结果 → 调触发器场景屏蔽接口（scope=本上电周期）→ 15-7拦截。接口清单（订阅/取消/callback/能力描述…）里目前没有这个"运行时屏蔽"，要提给刘杨。临时方案=上电周期1次，副作用：没拒绝只是没理的用户也永远失去本次推荐。
🟡 K4（待拍）"儿童"值域怎么圈：婴儿算不算亲子？
VLM值域=婴儿/儿童/青中年/老年。婴儿要的是摇篮曲不是互动问答。跟益红拍：条件配 等于[儿童]，还是 包含[婴儿,儿童] 且内容清单按年龄分流。
🟡 K5（待拍）儿童位置要不要约束
现在配的是"任一位置年龄=儿童"。要不要排除主驾区域（VLM误识成儿童的脏数据防线）？建议条件加 位置≠主驾区域，或依赖K8防抖兜底。
🟡 K6（待拍）频控语义三连问
①用户接受后，本次上电中途又上来一个小孩，还推吗？②3min失效没人理，条件仍满足，重试吗？③服务区下电再上电，重新推吗（"本次上电"字面=会重推）？——三问都拿去问益红，答案写进execute_policy配置和PRD。注意益红表里同类场景有两种口径："本次上电不再"（本案）vs"本次行程不再"（AI导演案）——频控维度要同时支持上电周期和行程周期两种作用域。
🟡 K7（待拍）配置平台动态条件区还是"vlm信号"占位
待改造点①：VLM条件的7字段（camera_group/seat/query/interval/timeout/priority/mutex_group）UI还没实现。0804链路打通用什么配"乘客列表.年龄=儿童"？确认过渡方案（后台直配JSON？），别让配置平台成为这个case的排期卡点。
🟢 K8（注意）年龄识别抖动
儿童↔青中年在边界年龄（12-15岁）可能抖，张兰若的glitch警告适用 → 15-6防抖：连续K帧一致才认，K值本案建议2（低频任务K=2即10分钟，太久——如果走了K1的oneshot方案，K帧按oneshot重试实现）。
🟢 K9（注意）3min失效是卡片系统的TTL，别揽
触发器的责任到push事件为止。评审时有人问"3分钟失效你怎么做"，答案是"卡片系统按ttl_min收走，触发器透传该参数、不实现倒计时"。
🟢 K10（注意）合规安全
舱内图不上云，链路里上云的只有标签JSON（乘客列表），合规OK。播报话术不点名不描述外貌（"小朋友也在车上"而非"后排的小女孩"），Director话术规范兜底。

#可挑战点核对结论
1"主驾入座"我选了端状态占座信号，而不是VLM的action=上车或车门信号三选一里占座最可靠（压力传感器，无视觉误识、无网络依赖），且是配置平台24类既有字段——符合你自己定的P0原则"端状态能拿到的先用"。VLM action=上车 留作交叉验证，不做主条件
2T3+6s的事件JSON是我设计的示意，不是已定协议字段结构基于15-10"标准事件打包"的PRD方向+图A"下游输入"概念，具体schema还没和刘杨/佳杰对过——这份JSON可以直接当接口定义初稿的素材，但对外讲时说"示意格式"

① 这个case的触发器部分只有两条条件+一条频控，真正的工程量在三个交界处：oneshot插队链（K1）、事件×状态AND语义（K2）、拒绝回流接口（K3）——三个都是通用能力，借这个case立项，后面30个"上车了"系列场景全部受益；
