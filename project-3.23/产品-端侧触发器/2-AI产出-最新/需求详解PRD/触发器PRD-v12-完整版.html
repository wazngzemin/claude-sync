一背景与产品定位

1.1 背景(承原PRD四点,仍然成立)

  1. 主动服务需求:用户不只需要"我说你做"的响应,还需要系统在合适时机主动服务——"车里热了自动开空调""10分钟后关座椅加热""副驾打瞌睡提醒降温"。这需要一个"条件监控+自动触发"的引擎。
  2. 条件任务增长:赛力斯条件任务清单已21+个场景且持续增加(7.2晚会议已过到20行),缺乏统一的条件注册、判定、执行引擎,每个场景各写一套逻辑不可维护。
  3. 端云协同需求:定时器必须在本地(弱网/断网不能停摆),VLM视觉监控必须在端侧(隐私+带宽+延迟),但复杂语义判定和冲突仲裁需要云端。纯云端或纯端侧都不够。
  4. Advisor感知订阅:动态Advisor需要"耳目"持续感知外界变化。感知订阅=注册触发器的tool,把Advisor的自然语言需求翻译成触发器的结构化注册参数。触发器是Advisor视觉感知的底座。

1.2 一句话定义

条件触发器 = 采集信号 → 判断条件 → 驱动执行。根据采集到的各类信号,当满足一定条件时,驱动下游执行特定动作。它自己不思考(不做语义判断、不出建议),只做一件事:把"持续变化的信号流"转换成"离散的、值得叫醒模型的事件"——用运算符挡住99%的噪音,让模型只处理1%值得处理的时刻。

1.3 端云分工铁律(6.22定版,持续有效)

#铁律
①动态注册经云端触发器中转(不绕过)——云端要有全局注册视图,否则冲突仲裁失效
②端结果→信号库→Context(不直推Advisor、不跳级)。v12补充:动态任务实时回传通道存在"原路返回"争议,见§12.1
③触发器只读信号库(绝不直接监听原始信号源→解耦)
④condition→Context供读取 / event→触发器组装push给Advisor(Advisor不轮询)
⑤不开发两个触发器:物理上端云两个部署,逻辑上一个判定权——同一条件绝不两边都判

二名词与运算符语义

2.1 核心名词

名词定义
信号 Signal各类条件对应的具体数据。传统=车身状态key-value(doorState:open);融合大模型后也包括模型可理解的自然语言(VLM对环境/人物动作的描述)
场景 Scene{一组条件+一组执行内容},即"if...then...":if舱内温度<10°且空调未开,then打开主副驾空调25°
条件 Condition信号key + 运算符 + 目标值(可选位置限定、有效窗口)
事件 Event条件命中后触发器组装的标准消息,push给下游(Advisor/Director)
SP / KV槽位SP=VLM每轮推理读的指令(看什么);KV槽位=每个动态任务在VLM返回JSON里的固定answer位,key=任务名称(西钺7.4口径)
AlwaysOnVLM常驻周期推理任务,官方三分类见§6.1
oneshot按需单次推理任务,插队执行(高优抢一个周期窗口,AlwaysOn轮空一次)

2.2 运算符语义(条件引擎的全部本事)

运算符盯什么触发几次示例
等于 / 不等于当前值==X(状态)满足期间持续为真"车里存在小狗就保持车窗全关"——有狗就一直约束
大于/小于/大于等于/小于等于数值比较持续为真电池电量 ≤ X%
包含 / 不包含数组型枚举持续为真weather_type 包含 [雾,大雾](天气可多值["下雨","浓雾"])
变更为 X从"非X"跨到"X"的那一刻(事件沿)穿越时一次"变更为 睡觉"——刚睡着那一下触发;挂着P档不再响
无 变更为 有出现(存在性正沿)一次"出现小狗告诉我"——狗刚上车响一次
有 变更为 无移除(存在性负沿)一次"小狗不见了告诉我"
变更(不指定)值变了(任意)每次变都触发日志/同步类,业务少用
事件类型→运算符映射(视觉标签表定版):非持续行为(比OK手势/打哈欠)=等于,出现一次给一次;持续行为开始(开始睡觉)=变更为[行为];持续行为结束=变更为[结束态];存在物品=等于+持续通知;出现物品=无变有一次;移除物品=有变无一次。所有跟时间相关的判断(出现/移除/开始/结束/持续多久)都不在VLM返回中,全部由触发器搞定(张兰若6.30定版)。

七紫线上传通道与异常处理(7.4评审范围)

7.1 上云通道官方口径

通道:由端侧触发器请求AI-Box,并接收结果,随后将结果以端状态协议透传给AI Service(丁彬官方文档)。要点:
① 紫线=VLM→端侧触发器→AI Service;蓝线(AI Service→云端端状态)归韩杰,不在本需求;
② 触发方≠接收方:原VLM SDK"谁触发谁接收",现改为"端侧触发器触发,但结果透传AI Service供全下游用"——需与斯华(意图SDK)+AI service链路拉通实现;
③ 上云范围:按丁彬格式表"是否通过端状态上云"列(舱内高低频=是、舱外=是,第10/13行动态查询结果除外);动态拼SP结果和舱内VQA不走端状态,谁调用谁拿(7.4结论);
④ 模型原始输出(一坨JSON)→上云前要解析拆分成独立信号(天气/路面拆开)——解析责任方待定(§12.9)。
