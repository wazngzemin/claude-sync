结构化信息（多字段/多属性对照、多并列项各自有固定属性）默认直接用表格呈现，不要先写大段文字。

**Why:** 用户是产品经理，需要一眼数清"有几个项"、对齐看每个项的属性。大段文字要通读才知道全貌，表格对照清晰、找信息快。2026-06-23 讲端侧信号库6字段（id/key/value/source/timestamp/ttl）时，我先用了几轮大段文字，用户明确指出"表格比文字好，你为什么还要用文字"。

**How to apply:** ① 讲"N个并列项 + 每项有多个固定属性"（如字段表、模块对比表、参数清单）→ 直接表格，因果逻辑塞进单元格；② 只有讲单一线性流程、单个因果推导时才用文字段；③ 表格要在单元格内写详细（用户要"详细"+"表格"，不是精简表格）→ 用 `<br>` 换行把要点铺开，看一个单元格就能完全理解那一项；④ 分类的大表用第一列分组（如①标识/②内容/③时效）。关联 [[feedback_fork_workflow]] [[feedback_apply_user_role]]。
