# 种子数据补 Context 分工会议

## 基本信息

- **会议主题**：种子数据补 context 分工
- **会议时间**：2026-04-01（周三）15:30 - 16:16
- **主要发言人**：郝晓伟（主讲）、张坤、李强、梁浩杰、罗丽、宋丛丛、赵磊、李华健

## 会议概要

本次会议主要讨论车内 AI 助手（Planner）**种子数据（seed data）的 Context 构建工作**，包括数据格式规范、模块分工和时间计划。种子数据将用于后续评测和数据泛化。

## 种子数据模块划分

根据 [[planner]] 当前包含的工具和能力，种子数据分为以下模块：

| 模块 | 内容 | 示例 |
|------|------|------|
| 像人一样对话 | 闲聊类 case | "你觉得宇宙中有外星人吗？" |
| 明确车控 | 多指令、强多轮、结合人名/视觉状态/媒体操作 | "空调调到20度开外循环" |
| 模糊车控 | 隐含意图的车控 | "车里好热"、"帮我调舒服点" |
| 人脸注册和记忆操作 | 认识人、记忆增删改查 | "认识一下坐在副驾的明骏" |
| POI 检索/路径规划/导航 | POI 搜索、多途经点、顺路规划、结合记忆/人人对话/联网信息 | "帮我规划最顺路的路线" |
| 内容和媒体搜推 | 音乐、短视频、播客、内容创作生图 | — |
| 信息查询和问答 | 车书知识、视觉问答（舱外/舱内）、天气、联网 | "前面那个牌子什么意思" |
| 主动服务（Advisor） | Advisor 主动建议 | "明骏上车了，调节舱内环境" |
| 跟进任务 | 定时任务、条件任务、GUI 长程任务、长时任务 | "10分钟后关掉座椅加热" |
| 高阶体验 | 地图画画、恐怖剧本杀、冥想场景 | "帮我规划一条爱心路线" |

## Context 字段规范

每条种子数据的 Context 由以下字段组成：

### 1. 视觉信息
- 时间、地点、天气
- 格式后续由仁杰统一梳理

### 2. 舱内情景描述（分静态+动态）

**静态内容**（高清图像识别）：
- 乘客和宠物列表、位置、外貌特征（发型、眼镜、衣物）
- 舱内物品（过道杂物等）

**动态内容**（多帧连续图像）：
- 乘客行为状态（开车、看手机、睡觉）
- 情绪状态（正常、疲劳）
- 舱内整体描述和氛围

### 3. 舱外情景描述（分静态+动态）

**静态**：天气、车辆所处环境
**动态**：人员活动（横穿马路、摆放锥筒）、红绿灯变化

### 4. 端状态
- 车窗状态、空调状态（温度/风量/前后排）、座椅、氛围灯等
- 按位置分别描述，用分号隔开

### 5. 用户记忆
- 饮食偏好、社会关系、地点记忆等
- 根据具体 query 关联必要的记忆内容

### 6. Goal List
- JSON 格式描述当前目标、目标细节和进展

### 7. 人音区映射
- 主驾/副驾/二排左/二排右 各位置坐的人名
- 未注册乘客标注为"未注册乘客"

### 8. 说话人字段（新增）
- 标注当前 query 的说话人是谁
- 与人音区映射和用户记忆做关联

### 9. 车辆知识
- 格式待定，后续由西钺确定后补充

## Context 构造思路示例

- **端状态差异化构造**：同一 query 构造不同端状态（如空调已是目标值 vs 低于目标值）
- **工具返回构造**：POI 返回中包含营业时间、停车信息
- **营业状态构造**：同一 query 分别构造正在营业/未营业两种工具返回
- **视觉描述构造**：根据 query 需要重点关注舱外前景描述

## 关键讨论

### 记忆关联机制
- 记忆与账号+FaceID 绑定：车上所有人共享对话产生的记忆
- 说话人可以查询记忆生成时在车上的所有人的记忆
- 后上车的人无法查到之前车上产生的记忆

### 数据质量保障
- 赵磊提出需要 review 机制
- 张坤确认写完后需逐条 review
- 优先保证每个模块都有 case 覆盖，评测流程先跑通
