子链路 A.2 · 充电桩开充电口盖 · v3（线标"为什么"）

v3 升级：每条线都写字——为什么 Yes / 为什么 No / 走这条线的原因 / 数据流含义。
对照丁彬图：他每根线上都标了"为什么这么连"，这是评审能讲清楚的关键。
v3 升级点：每条线都有带背景的文字标注，分三类语义——
· ✅ Yes 标注（绿）：写"为什么 Yes"——命中条件是什么。如"R→P 命中 change_to P"、"last=F → this=T 首次穿越"
· ❌ No 标注（红）：写"为什么 No + 为什么这么设计"。如"T→T 不重复触发，避免每5s周期都开盖"、"高风险动作需用户确认"
· 数据流标注（白/黄/蓝）：写"为什么走这条线 / 走的含义"。如"为什么原路返回：实时判定低延迟，绕 Context=10s 延迟（7.4 推翻）"、"为什么不经 Director：低风险动作免二次确认"

对照丁彬的图：他每根线上都标了"为什么这么连"——这是评审能讲清楚的关键。
