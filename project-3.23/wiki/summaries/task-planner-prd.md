## 概述

本文档是AI汽车1.0任务Planner的核心PRD，定义了任务拆解（单步、条件、多步、持续任务）、工具库、性能目标和准确率目标。Planner是连接用户意图和下游执行的核心模块。

## 关键章节

### 1. 任务拆解类型

| 任务类型 | 说明 | 示例 |
|----------|------|------|
| 单步单任务 | 一个简单指令 | "打开空调" |
| 单步多任务 | 一个指令包含多个操作 | "打开空调并播放音乐" |
| 条件任务 | 满足条件后触发 | "十分钟后关闭座椅加热" |
| 多步任务 | 需要多个步骤完成 | "播放我是歌手第四季夺冠的歌" |
| 持续任务 | 长时间持续执行 | "路上和副驾朋友聊聊天" |

### 2. 工具库
Planner可调用的工具包括：
- **车控工具**：空调、车窗、座椅、灯光等车辆控制
- **本地生活**：附近搜索、餐厅推荐等
- **音乐**：播放、搜索、推荐
- **记忆**：用户偏好、历史行为
- **条件任务**：注册和管理条件触发任务
- **联网问答/搜索**：实时信息查询
- **语音扮演**：角色扮演、语气模拟
- **Face ID**：人脸识别相关
- **VQA**：视觉问答
- **状态查询**：车辆状态查询

### 3. 性能目标
- 首字响应P50 <= 500ms
- 首字响应P90 <= 800ms

### 4. 准确率目标
- 推理ACC >= 90%
- 回复ACC >= 93%

## 核心要求

- 任务拆解需逻辑合理、下游可承接
- 复杂车控需对齐0.5标准（ACC >= 90%，控制完成P90 <= 2000ms）
- 条件任务需支持定时器和事件触发
- 多步任务需有清晰的步骤划分和过渡话术
- 持续任务需能维护长期状态

## 提取的实体

- **Planner**：任务规划和拆解核心模块
- **工具库**：Planner可调用的下游能力集合
- **条件任务**：基于时间或事件触发的任务类型
- **持续任务**：需要长期维护状态的任务类型

## 提取的概念

- **任务拆解**：将用户自然语言指令分解为可执行步骤的过程
- **首字响应**：从用户说完到系统开始回复的时间延迟
- **推理ACC**：Planner推理结果的正确率
- **回复ACC**：最终回复内容的正确率
