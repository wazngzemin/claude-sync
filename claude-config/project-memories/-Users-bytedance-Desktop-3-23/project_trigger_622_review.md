2026-06-22 端侧触发器需求评审核心结论（已并入 触发器PRD-v2.html §11.5，PRD升v2.1）：

1. **端侧 scope 大幅收窄**：固定能闭环的功能（起步/上下车安全、睡眠提醒、无感迎宾、开车门、充电大师开充电盖）全回**赛力斯 CDC 底软**自己做，不走触发器。端侧触发器只保留 { 云端下发的动态/定时任务 + 可运营临时任务 + 动态视觉感知事件 + 通用视觉通道 + 任务中心端云同步 }。仁杰判定标准：常驻固定标配/key-value强字段→CDC；有生命周期/动态/可变/可运营→触发器。西钺：不开发两个触发器（打架+浪费），端侧触发器只在CDC，VLM在AIBox。

2. **状态 vs 事件链路纠错（佳杰当场纠正泽民）**：Advisor 订阅"事件"等 push，**不轮询 Context**。原图 ㉑Context(21)→⑰Advisor(17) 错 → 正：⑮云端触发器(15)→⑰Advisor(17)。状态/condition走Context；事件由⑮监听信号变化、组装后push。端侧不能push，端模输出同步到信号库，云端有"做事件的模块"持续关注端侧信号库。端模只报客观状态("坛子当前在地上")不做是否判断。

3. **动态视觉感知事件链路（仁杰修正）**：Director不直接到触发器→先建动态Advisor→拆解改写注意力SP("当X出现就告诉我")→调⑮注册→下发端模观测→⑮组装事件返回⑰推理。**3卡点7月中定**：①SP下发路径 ②端模能否做条件式注意力"看到X才上报"(没试过,需prompt+有限集+改端上prompt渠道) ③视觉判定放哪。佳杰主张所有视觉trigger统一走云端触发器。

4. **通用视觉事件/端侧泛化感知通道**：兜底长尾，透传动态query给AIBox，分舱内静/舱内动/舱外三类，prompt增量(非覆盖)。

5. **任务中心端云打通(佳杰新依赖)**：泽民建端侧持久化任务库+"任务级vs感知级"字段+enable/disable反向写入接口；佳杰只从端侧取数据。

6. **调度归属**：任务调度在佳杰AIBox(优先级排序)，不在触发器。施佳杰=佳杰。VLM:舱内兰若/舱外丁彬。

7. **丁彬端模能力三层评估(40:10-41:27)**：a)当前只做客观观测("看啥"可以，"看到就做Y"后半步做不了) b)有限集内prompt限制应该行，发散不确定——没测过 c)"prompt有渠道改到端上就OK"。

8. **西钺端模输出规范(46:12-47:38)**：纯客观状态("坛子在地上")不做是否判断；Context存时间序列list。物体+位置场景(小孩在前排)复杂度高不一定搞得定。需在端状态**新增动态观测字段**，与AlwaysOn固定输出分开。

9. **赛力斯可穷举清单状态**：泽民上周让赛力斯梳理但未产出——赛力斯在群里拉了十几人无人响应。

**原子重构（PRD已落地，非仅批注）**：云端⑮ 12→15(新增15-13状态→事件push / 15-14动态视觉注意力SP注册下发 / 15-15通用视觉通道)；端侧㉔ 12→9(删除移交CDC的5个:24-3信号扫描/24-4两级判定/24-6防抖/24-7仲裁/24-10阈值穿越；保留24-1/2/5定时器/8'视觉SP下发/9五态/11上报/12'下发任务执行；新增24-13任务中心同步/24-14通用视觉通道)。核心=24-5本地定时器(唯一不可被CDC替代)。
