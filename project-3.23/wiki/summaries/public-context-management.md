## 概述

本文档定义了AI汽车1.0的公共Context管理服务，统一管理多模块共享的上下文信息，包括情景描述、对话历史、事件日志、车辆状态、屏幕UI、用户记忆、任务状态、基础联网信息、豆包探索、通用任务经验、工具知识、专有知识和豆包人设。

## 关键章节

### 1. Context类型

| Context类型 | 说明 | 来源 |
|-------------|------|------|
| 情景描述 | 舱内外环境、成员状态 | 端侧感知模型 |
| 对话历史 | 用户与AI的多轮对话记录 | 对话系统 |
| 事件日志 | 人物、车辆、AI的行为记录 | 各模块上报 |
| 车辆状态 | 空调、车窗、座椅、能源等 | 车辆CAN信号 |
| 屏幕UI | 当前屏幕显示内容 | HMI系统 |
| 用户记忆 | 用户属性、偏好、行为 | 记忆模块 |
| 任务状态 | 当前任务列表及状态 | Planner |
| 基础联网信息 | 天气、节日等 | 云端服务 |
| 豆包探索 | 主动探索和推荐 | 探索模块 |
| 通用任务经验 | 常见任务处理经验 | 知识库 |
| 工具知识 | 各工具的能力描述 | 工具注册 |
| 专有知识 | 车书、产品手册等 | 知识库 |
| 豆包人设 | AI性格特征 | 配置 |

### 2. Context压缩与过滤
- 针对长上下文场景的压缩策略
- 根据当前任务过滤无关Context
- 优先级排序，保留最关键信息

## 核心要求

- Context需实时更新，保证时效性
- 多模块共享需保证数据一致性
- 压缩和过滤不能丢失关键信息
- 长上下文场景需控制token消耗

## 提取的实体

- **Context服务**：统一管理上下文的中间件
- **情景描述**：舱内外环境和成员状态的文本描述
- **事件日志**：按时间线记录的行为日志
- **用户记忆**：用户画像和偏好数据

## 提取的概念

- **公共Context**：多模块共享的统一上下文空间
- **Context压缩**：减少上下文长度同时保留关键信息的机制
- **Context过滤**：根据当前任务筛选相关上下文的能力
