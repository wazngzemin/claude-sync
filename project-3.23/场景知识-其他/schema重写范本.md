# 场景知识 新 Schema 重写范本

> 范本对象：R10 车内小憩（DRV-01-01-09）+ R14 后排勿扰（DRV-01-01-13）
> 这两条在原 schema 下相互交叉、难以区分。
> 在新 schema 下，通过【触发样本】+【互斥锚定】实现 RAG 召回的精准 disambiguation。
> 本文档作为剩余 11 条场景重写的写作模板。

---

## 一、新 Schema 字段定义

| 字段 | 必填 | 用途 | 写作规范 |
|------|------|------|----------|
| 序号 | 是 | 唯一索引 ID | DRV-01-01-XX 格式 |
| 一级 / 二级 / 三级 | 是 | 人类目录树 | 不用于 RAG 检索，仅用于文档导航 |
| 四级 | **否（默认删）** | 仅在确有子细分时保留 | 例如观影下面的"前排/后排/5D" |
| **触发样本** | 是 | RAG 召回**正向锚点** | 5-10 条具体 query；覆盖主流 + 长尾 + 隐式三类 |
| **互斥锚定** | 是 | RAG 召回**反向锚点** | 3-5 条易混淆 query + 应该走的目标知识 ID |
| **场景要点** | 是 | 区别于相邻条的核心判断 | 不超过 200 字，一句关键差异 |
| 执行逻辑 | 是 | 被召回后模型怎么做 | 保留原写法（步骤化 + 决策规则） |
| 对话示例 | 是 | few-shot 学习样本 | 保留原写法（带括号注释） |

## 三、范本 2：R14 后排勿扰（DRV-01-01-13）

### 一级 / 二级 / 三级
用车关怀 / 车内关怀 / 后排勿扰

### 四级
（删除——原文"后排勿扰、后排成人休息勿扰、后排儿童看护勿扰、后排状态实时查看"是子情况枚举，已被执行逻辑覆盖，作为 schema 字段不承载额外信息）

### 触发样本

```
主流说法：
- "乐乐睡了后面安静点"
- "孩子睡着了别打扰"
- "把后排音乐关小点别吵到他"
- "后面那个屏幕关了"

长尾说法：
- "他在睡你们小声点"
- "锁一下后排按键防他乱按"
- "看看后面睡得怎么样"
- "后面别响"

隐式说法：
- 后排儿童闭眼歪头 + 主驾说"安静点"（视觉 + 语音联合触发）
- 后排音量 > 15 + 后排有人睡眠迹象（advisor 触发）
```

### 互斥锚定

```
- "我想睡一会儿"（说话人主驾或副驾 + 驻车）        → DRV-01-01-09 车内小憩
- "孩子哭闹哄哄他 / 给他放点东西"                  → DRV-01-01-01 陪伴安抚
- "帮我冥想 / 放松一下"                            → DRV-01-01-08 冥想空间
- "后排冷 / 热 / 调一下温度"（主动调节后排舒适度） → DRV-01-01-04 急速调温（合并后）/ 走相应舒适用车场景
```

### 场景要点（瘦身后）

前排用户表达**后排有人需要安静**（睡眠/休息/儿童看护）——核心是**抑制后排存在感**（关后排音乐 + 关灯 + 锁按键 + 静音 TTS 播报）+ **被动安全看护**（摄像头查看 + 异动监测 + 不打扰式提醒）。
区别于车内小憩：本条服务"别人睡"，后者服务"自己睡"。
区别于陪伴安抚：本条是**降低后排存在感**，后者是**给后排注入内容**——执行逻辑完全相反。

                                          ↑

  共享触发词："睡 / 困 / 安静 / 小声 / 勿扰"
  靠【说话人位置 + 车辆状态 + 视觉信号】区分

**关键洞察**：相邻知识共享大量触发词，**仅靠 query 文本无法稳定 disambiguate**。互斥锚定字段必须显式写出"在什么条件下走另一条"，让模型同时召回多条时有依据做选择。

## 五、写作 Checklist

每条场景重写完后，对照以下条目检查：

- [ ] 删除"四级"字段（除非确有独立子细分）
- [ ] 触发样本 ≥ 5 条，覆盖主流 / 长尾 / 隐式三类
- [ ] 互斥锚定 ≥ 3 条，每条都指向具体的目标知识 ID
- [ ] 互斥锚定的目标 ID 必须真实存在于本 xlsx 中（避免悬空引用）
- [ ] 场景要点 ≤ 200 字，且必须包含"区别于 X 条：……"的对照句
- [ ] 执行逻辑里的"做事原则"（如"全程禁止 X"）保留；"用户需求理解"类的冗述删除
- [ ] 对话示例的括号注释保留——这是给模型学"为什么这样回复"的关键信号
- [ ] 跑一遍假想 RAG：用触发样本里的每条 query 推演一遍，应当稳定召回本条
- [ ] 跑一遍互斥锚定：用互斥锚定里的每条 query 推演，应当走到指针指向的目标条而非本条
