7.6下午VLM链路定版 7.6晚事件融入DT内审
双通道分离（7.6定版）
• 控制通道（橙色）：触发器 ↔ VLM SDK，触发指令 + 状态回调
• 数据通道（蓝色）：VLM SDK → 数据中心，原始输出直送，触发器不经手
• 原因：触发器是条件判断模块，不该承担数据透传；VLM数据量大频率高，经触发器中转增加延迟
关键结论（7.6各方确认）
• 触发器保证"请求发送成功"，VLM SDK保证"收到后执行成功"（职责切割）
• AlwaysOn双通道已定版；OneShot回传路径待评审
• 数据中心整包透传，按需缓存；端云规则必须一致（汪帅原话："否则就是bug"）
事件融入DT（7.6晚内审方向）
• 事件 = 异步消息（手动操作/GUI/工具反馈），不只是trigger
• MVP：最近1条重要事件插入对话历史（解决QQ音乐登录等bug）
• 长期：情景注意力模块（事件+VLM+记忆 → 自然语言情境描述）
