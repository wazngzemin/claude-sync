用户(泽民/AI PM)被领导(大马)指派接手 **触发器(Trigger) 产品**——它是 主动服务/静态Advisor、动态Advisor、条件任务、机器人 四方共用的底层"事件感知+判断+触发"引擎,三要素=触发器/条件(Condition)/动作(Action)。

**整条链路:** 用户语音→ASR→三层漏斗→Planner→意图模型(把人话翻成触发器协议,依赖"高频条件清单")→云端触发器(注册动&静、冲突仲裁、承载模型生成事件)→端侧触发器(判断是否触发、定时器、执行策略)→读端状态信号库v2.5→命中触发→notify给云 或 端上直接执行→tool_feedback回传。现状:**所有任务都走云端Planner**;目标态(赛力斯阶段):端侧直接闭环+端云分工+云端接模型。

**关键负责人:** 意图=廖婉婷;触发器开发=刘洋(北京);测试=萝莉/绿货;后台配置=叶芳;端状态信号库v2.5维护=韩天亮;context=仁杰;动态Advisor产品=施佳杰;主动服务/静态Advisor产品=赵益红;机器人=吕壮;后台账号权限=找文涛/芭比(BABA)。

**眼下P0(上汽DNA,610交需求,6/15前评审):** ①整理一版标准《高频触发条件清单》(对标理想任务大师,从信号库v2.5精选)→②交意图调模型→③评估赵益红事件表的"?"。协议改造/任意位置/视觉/端侧执行/时间/距离事件 全部挪到赛力斯阶段(仁杰拍板)。
