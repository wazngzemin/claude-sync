## 概述

本文档记录了任务规划与意图理解模块的prompt工程实验，定义了XML结构的prompt模板，接受用户query、子Agent建议、情景描述、事件日志、豆包人设、车辆状态、用户记忆、当前任务状态和基础联网信息，输出回复和任务队列JSON。

## 关键章节

### 1. Prompt结构

```xml
<用户Query>{{用户语音输入}}</用户Query>
<其他智能体建议>{{其他智能体给到管家的建议}}</其他智能体建议>
<情景描述>{{情景描述}}</情景描述>
<事件日志>{{事件日志}}</事件日志>
<豆包AI的人设>{{豆包人设}}</豆包AI的人设>
<端状态>{{端状态}}</端状态>
<用户记忆>{{用户记忆}}</用户记忆>
<当前任务状态>{{当前任务状态}}</当前任务状态>
<基础联网信息>{{基础联网信息}}</基础联网信息>
```

### 2. 输出格式

```xml
<回复>[需要立刻回应用户的话]</回复>
<任务队列>[新的任务队列JSON]</任务队列>
```

任务队列JSON格式：
```json
{
  "change_mode": "add/refine/del",
  "task_id": "任务编号",
  "rank": "执行次序",
  "task_content": "任务具体内容",
  "task_condition": "触发条件",
  "task_nlg": "执行时对用户说的话"
}
```

### 3. 实验案例

#### 实验1：接受子Agent建议
- 输入：空用户query + 多Agent建议（情感陪伴、舒适空间、省心出行、内容消费）
- 输出：自然回复 + 多个新增任务（座椅按摩、导航规划、内容推荐）
- 性能：首token 1.2s，总耗时8.8s

#### 实验2：接受用户Query
- 输入：用户闲聊query + 空Agent建议
- 输出：自然闲聊回复 + 空任务队列
- 性能：首token 1.2s，完整耗时3.6s

## 核心要求

- 其他Agent建议不一定非要采纳
- 闲聊类话语仅回复，不调整任务队列
- 可迅速做完的任务可插队优先
- 正在做的任务如还需做，被插队时调整rank到后面

## 提取的实体

- **子Agent**：提供建议的垂直领域智能体
- **任务队列**：待执行任务的优先级列表
- **change_mode**：任务队列的调整模式（新增/变更/删除）
- **task_nlg**：任务执行时的自然语言播报

## 提取的概念

- **Prompt工程**：通过优化输入提示提升大模型输出质量的技术
- **XML Prompt结构**：使用XML标签组织多模态输入的prompt设计模式
- **任务插队**：高优先级任务插入当前队列的机制
