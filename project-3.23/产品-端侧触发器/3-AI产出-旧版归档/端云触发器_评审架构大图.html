11 云端触发器 / 规则执行引擎

承接动态任务和系统预设场景。只做确定性规则执行，不做开放式语义决策。
11.1 场景库静态规则、动态任务、系统预设、版本灰度。
11.2 规则扫描信号更新触发、周期扫描、任务状态过滤。
11.3 条件判定AND/OR、group、range、变更为、持续时长。
11.4 去抖冷却debounce、cooldown、max_execution、expire。
11.5 优先级仲裁P0-P4、重复抑制、互斥组、冲突检测。
11.6 端云分类云端处理、下发端侧、VLM调度、本地兜底。
11.7 标准事件task_id、原始任务、快照、是否二次推理。
11.8 Callback RouterDirector、动态Advisor、Context、任务中心、端侧。

12.2 动态 Advisor一目标一实例；条件、定时、VLM、GUI 闭环。

触发器模块到底要做什么

模块 必须做 明确不做 评审输出
云端触发器 接收注册；扫描云端信号库；执行条件判定；去抖冷却；优先级和冲突仲裁；生成标准触发事件；callback 路由。 不做开放式语义推理；不直接控车；不和某个 Advisor 实例强耦合。 标准事件、触发记录、任务状态、失败原因。
端侧触发器 Core 端侧定时器；弱网/断网 AI PUSH；本地任务持久化；本地状态组合；网络恢复后同步。 不替代 Director；不做复杂语义仲裁；不绕过安全策略做车控。 本地触发、倒计时、离线履约、恢复同步。
VLM 调度器 视觉订阅；SP/attention；图片获取；VQA 调用；阈值穿越；P0-P4；互斥组；终态回写。 不承接用户意图理解；不把 timeout 写成无异常；不无限并发视觉任务。 VLM JSON、视觉事件、success/empty/failed/timeout。
Callback Router 按任务归属投递到 Director、动态 Advisor、Context、任务中心、端侧。 不做业务推理。 可靠投递、失败重试、投递日志。

研发评审必须确认

DSL 能力是否支持 =、>、<、range、变更为、持续时长、group logic。
信号范围韩天亮信号库 v2.5 哪些可量产，哪些只是规划字段。
端侧定时器是否支持持久化、上下电恢复、任务中心倒计时。
VLM 并发是否固定 1 个，还是按 VLM_CABIN/VLM_FRONT 分组。
图片服务车内、车外、屏幕 image url 是否具备稳定接口。
callback是否有 ACK、重试、幂等和失败日志。
端侧闭环白名单提醒类可本地闭环，车控类是否必须回 Director。
CDC 边界哪些赛力斯自己闭环，哪些事件给字节触发器。
