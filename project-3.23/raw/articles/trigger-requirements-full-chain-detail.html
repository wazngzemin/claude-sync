《协议范围说明》里必须包含这些块：

区块写什么例子
支持范围哪些条件可解析（指向需求1的20~30条）温度/车速/车门/座椅…
取值类型映射每类条件对应哪种取值方式温度类→数值阈值；车门类→布尔/变更为
协议格式一条条件任务的结构长什么样condition{signal,op,value} + action
正例用户话 → 期望解析结果"超25度开空调"→{temp>25, 开空调}
反例 / 不支持明确划出现在做不了的，免得模型乱猜"我有点冷"（模糊，本期不支持）
边界规则多条件、否定词、时间词怎么处理"开空调但别太冷"→怎么拆
✅ 标准就是：意图同学拿着这份说明，不用回来反复问你"这个支不支持、那个怎么解析"，自己就能开工调模型。
B · 廖婉婷拿到后，怎么处理？

① 锁边界拿《协议范围说明》当约束 → ② 造样本用你给的正反例扩成训练/测试集 → ③ 调模型让它把用户话解析成触发器协议 → ④ 自测跑解析准确率、对齐格式 → ⑤ 回工期给你"调试要 X 天" → ⑥ 联调和你抽样话术，错的回炉。

可验收：610 能明确验收"用户说条件话 → 正确建成条件任务"。
能倒排：你拿到"调试 X 天"，就能倒推清单必须多早交，掌控节奏。
能回炉：解析错的条目有正反例做依据，补清单描述、再训，有据可依。
