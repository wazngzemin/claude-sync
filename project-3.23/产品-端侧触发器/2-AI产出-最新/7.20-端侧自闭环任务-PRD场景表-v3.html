核心定义：端侧自闭环 = 触发条件识别、决策、执行、反馈全部在车端完成，触发器直接调用车控，不上报云端。

v3 核对后的三个硬结论（与 v2 的差异）：
① 驾驶模式枚举里没有「湿滑」——driveModeState 全部33个枚举逐一核对，无湿滑/雨天模式。雨天湿滑场景的执行动作本身不存在，需产品换动作或车控确认有无未上表的模式。
② 「雪地」模式在枚举里明确存在（另有「混动雪地」）——雪天场景执行端支撑，卡点只在降雪识别。
③ 全簿搜索0命中的信号：后视镜加热、洗车模式、湿度、雨量、雾天/能见度、降雪/结冰——这6类信号在信号中枢任何一张表里都没有。

一、状态查询逐场景核对结论（本次核心）

场景需要的状态查询信号状态查询表里有没有具体信号与取值单项结论
1. 雾天后雾灯 雾天/能见度识别没有「能见度」仅出现在后雾灯的描述文字里，不是信号识别缺失
车速有vehicleSpeed（车辆速度）：0~655.35 km/h可用

潜在替代：舱外视觉有但空白outCabInfo（舱外信息）：视觉描述，自由文本，参数空白理论上视觉链路可识别雾天，需找丁彬确认
2. 雨天湿滑模式 下雨识别可代理wiperState（雨刮状态）：关闭/1档/2档/3档——「在刮刷」=非关闭；独立雨量传感器信号没有可用雨刮代理，误触发风险高
车速有vehicleSpeed：0~655.35 km/h可用
驾驶模式（读当前模式）有driveModeState（驾驶模式）：33个枚举可读
湿滑模式（执行目标）枚举里没有33个枚举：雪地/超级节能/赛道/运动/舒适/沙地/泥泞/岩石/无极/动力/混动/纯电/越野/怡静/智享/澎湃/自适应/纯电经济/纯电舒适/混动智能/混动运动/混动雪地/混动越野/混动个性/纯电增程/智能电混/增程/性能/默认/经济/均衡/个性化——无湿滑执行动作不存在，场景按原设计无法落地
3. 雪天雪地模式 降雪/结冰识别没有全簿无降雪/结冰信号；只能车外温度+雨刮弱推断，冬季晴天低温会误触发识别缺失
车外温度有但值域空白ambientTemp（车外温度）：信号在，参数值域未填可用，值域需韩杰补
雪地模式（执行目标）枚举里有driveModeState 枚举含「雪地」「混动雪地」执行动作存在
4. 中低速后视镜加热 雨刮刮刷有wiperState：关闭/1档/2档/3档；「持续30s」需触发器引擎计时（信号本身无时长）可用
车速<70有vehicleSpeed：0~655.35 km/h可用
档位≠P有VehDriveState：D/P/R/N可用
后视镜加热（执行+反馈）没有全簿搜「后视镜」0命中，无状态信号、无控制信号触发条件全齐，卡在执行端
5. 洗车退出后视镜加热 洗车模式/传送带洗车没有全簿搜「洗车」0命中识别缺失
后视镜加热（执行+反馈）没有同场景4执行缺失
6. 湿气重天气 空气湿度没有全簿搜「湿度」0命中识别缺失

车外温度有但值域空白ambientTemp（车外温度）：同上可用
后视镜加热（执行+反馈）没有同场景4执行缺失
一句话总结：6个场景里，触发条件端被状态查询表完整支撑的只有场景4（中低速后视镜加热）；场景3执行端支撑但识别端缺；场景2出现「执行动作不在枚举里」的硬伤；场景1、5、6识别端缺关键信号。后视镜加热信号缺失同时卡死场景4、5、6的执行端——这是单点最高优先级缺口。

二、术语说明

术语说明
端侧自闭环触发条件识别、决策判断、执行动作、状态反馈全部在车端完成，触发器命中条件后直接调用车控接口执行，不上报云端。
条件任务任务中心中的循环任务，开关在「待办」区常驻，条件满足即触发，用户可启用/禁用。
触发器引擎监听端信号变化，按配置规则判断条件是否命中，命中后直接调用车控执行。
