一、端侧触发器要做的任务

⚠ 6.22 评审修正：下面 7 类是评审前设想——②③ 固定闭环类大半归赛力斯 CDC、④ 重塑为"动态视觉感知事件"。端侧实际只保留 { 定时器 + 动态视觉事件 + 通用视觉通道 + 任务中心同步 }。详见文首红框。

判断"该不该放端侧"只有一把尺子——这件事踩没踩到 4 条硬约束的任意一条：① 实时性（差几百毫秒~几秒就出问题）② 断网可用（隧道/地库/高速弱网）③ 隐私+带宽（画面/高频信号不能往云传）④ 安全（差几秒就是事故/人命）。踩到任一条 → 必须端侧。

⑤ 弱网/断网降级执行已下发端侧的规则在断网时本地判定+本地执行，不停摆纯云端方案断网=整条链路停摆；而进隧道/地库是高频日常云端定义上做不了（断网=不可达）。依赖：金龙(端侧触发器)产品定义《哪些规则需弱网闭环》（标在 15-9 下发时）+ 降级矩阵
⑥ 信号采集标准化③ 传感器/CAN 信号 → 标准化 → 写入 ④ 端侧信号库采集本身只能在端；标准化若放云端则每条原始信号都上传（带宽+延迟），且车型 CAN 差异传到云端才处理信号源物理在车上。依赖：CDC 研发（两套 CAN 信号清单）找 CDC 要上汽 DNA + 赛力斯 CAN 信号总表 → 产品做《信号-条件-可配置性 mapping》（也是 15-1 的输入）
⑦ 结果回写+上报结果写 ④ → 上报 ㉑ Context；断网存本地缓存队列、按时间戳补传（24-11，铁律②）没有端侧缓存 → 断网期间执行记录丢失 → 端云状态不一致缓冲层只能在端。依赖：施佳杰（Context 写入接口）产品出《上报事件 schema + 脱敏粒度》，找施佳杰对齐 ㉑ 写入接口
一句话：端侧做"快+笨+可靠"——定时、数值比较、白名单车控、视觉检测、安全告警、弱网闭环、信号采集回写。不需要脑子但要快、要断网也能干。语义理解 / 跨任务仲裁 / 推理一律不碰（归云端）。

看图三个要点：① ㉕ 感知器在触发器框里——VLM 不是独立模块，是触发器的感知半；② 所有结果先进 Context，再被 Advisor/Director 消费（橙→蓝→紫，铁律②，没有任何直推）；③ 只有一条红虚线（Director 一次性 VQA）绕开触发器，其余视觉全在触发器里。

四、VQA 和它们的关系

VQA = 云端一次性视觉问答（给一张图 + 一个 prompt → 拿一个答案）。它和 ㉕ 感知器是"看"这件事的两条底层路径，关系要单独讲清，因为它有一条是绕开触发器的。

4.1 VQA 的两条路径：一条经触发器，一条不经
