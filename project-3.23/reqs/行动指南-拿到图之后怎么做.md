## 2. 今天 / 本周：立刻做的 5 件事（按这个顺序）

> 原则：先抢"原料"（缺了开不了工的东西），再做"产出"，最后才谈"排期"。

### ① 今天上午——抢原料（不抢，下午就开不了工）
- **找韩天亮要《信号库 v2.5》最新版**，并**问清楚它锁不锁版本**。
  - 为什么先做这个：需求1、需求3 都靠它当字典。信号库一直在变，你必须钉死一个版本号再动手，否则做到一半底层变了，全废。
  - 话术见 §4 的"找韩天亮"。
- **找文涛 / 芭比开《后台配置器》登录权限。**
  - 为什么：清单最终要录进后台才"系统可用"。权限审批有时滞，今天不发起，到时候卡你。

### ② 今天下午——起需求1 的草表（你的核心产出）
- 新建一张 Excel / 飞书表，**一行一条件**，列固定为这 6 列（直接照搬图里的字段，别自创）：

| 条件名 | 取值判断方式 | 信号库v2.5 编号 | DNA能否支持 | 归属任务类型 | 备注 |
|--------|--------------|-----------------|-------------|--------------|------|
| 车内温度 | 数值阈值（>25℃） | （填编号） | ✓ | 舒适 | |
| 车门 | 布尔 / 变更为（从关→开） | | ? | 安全 | 套娃结构，见需求6 |
| 座椅占用 | 布尔 | | | 安全 | |

- **取值判断方式只有三种**，背下来，跟谁都这么说：
  - **布尔开关**（开/关，如座椅占用）
  - **数值阈值**（>N，如温度、车速）
  - **变更为 / 从无变有**（状态切换，如车门 关→开）
- **范围 20~30 条，对标"理想任务大师"，精选高频，不要贪全。** 初级 PM 最容易犯的错就是想做全——图里专门写了"不贪全"。

### ③ 本周内——并行做需求3（益红事件表评估）
- 拿益红那张《主动服务事件表》，对每个标 **`?`** 的事件给一个**准信**：能做 / 回掉 / 视觉排除。
- **判定规则极简，背下来**：
  - 在信号库 v2.5 里**查得到端信号 → 能做**
  - 查不到 → **回掉**
  - **强依赖视觉 → 直接划掉**（DNA 端侧视觉只支持 Face ID，别的视觉都做不了）
- 结论**回填进益红的表**，每条附一句理由（对应哪个信号 / 为什么不行）。

### ④ 明天——面聊吕壮（需求4，收料，不排期）
- 地点：漕溪北路。**只问两件事**，别多问：
  - ① 机器人会产生哪些事件？
  - ② 期望触发器怎么接（端侧还是云端、什么格式）？
- 拿到他的《机器人事件表》后：**归档，打标"赛力斯阶段"，不进上汽610**。这件事今天就是"约时间+列两个问题"，半小时的活。

### ⑤ 清单初稿出来后——立刻找廖婉婷算工期（需求2）
- 把定稿清单 + 一份"协议范围说明"（支持哪些条件、哪些取值、什么格式）发给意图团队。
- **核心动作：问她"模型调试要几天"**，然后**倒推**——她要 X 天，你就必须在 `6/15 - X` 之前把清单交她。这个倒推时间，就是你需求1 真正的 deadline。

## 4. 跟研发 / 各方沟通：怎么说才专业、不露怯

> 这是你最担心的部分。先记住一个心法：**初级 PM 不是靠"懂技术实现"显得专业，而是靠"把边界和接口说清楚"显得专业。** 你不需要会写代码，你需要会问对问题。

### 4.1 四条心法（贴在显示器上）

1. **永远用"输入→输出→工期"三段式说话**，不要说"你去做个 X"。
   - ❌"刘洋你把触发器改一下" → 像外行下命令
   - ✅"我这边产出的是这版条件清单（输入），我希望最终效果是用户说'超25度开空调'能被建成条件任务（输出），你看你这边大概要几天（工期）？" → 像专业 PM
2. **用系统自己的黑话**，别用大白话。研发听到你会用这些词，就默认你懂：
   - 端侧 / 云端、**状态变更 vs 当前态**、**协议拍平**、信号库、**触发时机/瞬间**、布尔/数值阈值/变更为、端云分工。
3. **不懂实现就大方说"实现你定，我对齐边界"**，这不丢人，反而专业。
   - ✅"具体怎么实现你们定，我只确认一件事：这个判断是放端侧还是云端？"
4. **每次对话结尾一定收三件事**：① 前置齐了没 ② 工期几天 ③ 验收标准是什么。收齐了再走。

### 4.2 分人话术模板（直接抄）

**▸ 找韩天亮（要信号库）**
> "天亮，我在做触发器的高频条件清单，需要拿《信号库 v2.5》最新版当字典。能不能给我一版、并帮我确认下这个版本号近期会不会再变？我想锁定一个版本开工，免得做到一半底层信号变了。"
（专业点：主动提"锁版本"，研发会觉得你懂工程节奏。）

**▸ 找廖婉婷（意图，需求2）**
> "婉婷，这是定稿的高频条件清单和协议范围说明。我希望的最终效果是：用户说'超25度开空调'，模型能解析成『条件:温度>25，动作:开空调』这样的触发器协议。
> 想跟你确认两件事：① 这个清单作为约束够不够、有没有你觉得解析不了的；② 模型调试大概要几天？我要拿这个工期倒推我交清单的时间，赶 6/15。"

**▸ 找刘洋（端侧研发，需求5/6/7/8/9 都他）**
> （需求9 端云分工）"刘洋，赛力斯这版触发器我们想做端云分工：云端管注册和冲突仲裁，端侧管判断触发和直接执行，冲突在云上消解完丢回端、端侧'没脑子只执行'。想先跟你对一下——目前端侧 SDK 能不能直连车控/TTS/播报？这块是后面定时器、距离事件的地基。"
> （需求7 定时器）"'10分钟后关座椅加热'这类，倒计时必须做在端上，云上跑会被隧道、弱网搞挂。你看把'时间'当成端状态的一个维度、定时器挂端侧，可行吗？大概工作量多大？"
> （需求6 协议拍平）"现在车门是套娃结构、position 当参数，导致'任意门开且X'配不出来。赛力斯版想把每个门拆成独立一级条件、position 和端状态拍平，再加个'任意位置'表达式。上汽这版我们先用模型把'任意位置'解析成具体几个门顶过去，不动协议。你帮我评估下真改造的工作量、要不要供应商配合？"

**▸ 找赵益红（主动服务，需求3）**
> "益红，你那张事件表里标 `?` 的，我按信号库 v2.5 逐条评了：有端信号的标'能做'，没有的'回掉'，强依赖视觉的直接划掉（DNA 端侧视觉只有 Face ID）。结论我回填进你表里了，每条都附了理由。你看着定这版主动服务上哪些事件、配成预设事件。"

**▸ 找萝莉（测试）**
> "萝莉，这批条件上线后要麻烦你找车实测，重点标两类：① 哪些真能触发、哪些其实没部署；② 上车/下车这种容易混的场景。你需要我提供什么样的用例清单格式？"

**▸ 找文涛 / 芭比（权限）**
> "想申请下后台配置器的登录权限，触发器条件清单要录进去上线。走什么流程、需要我提供什么？"

**▸ 面聊吕壮（机器人，需求4）**
> "壮哥，今天就想了解两件事：① 机器人会产生哪些事件；② 你期望触发器怎么接——走端侧还是云端、什么格式。我先收料归档，这块归赛力斯阶段统一规划，不占你上汽610 的节奏。"

### 4.3 三个万能句式（任何研发对话都能用）

- 对齐边界："**这块我只确认放端侧还是云端，具体实现你定。**"
- 要工期："**你大概要几天？我要倒推我的交付时间。**"
- 确认前置："**开工前你还缺我什么？我先给你备齐。**"

### 4.4 千万别说的话（会瞬间露怯）

- ❌"这个很简单吧，应该很快"——研发最烦 PM 估工期。
- ❌"你随便弄一下就行"——显得你不知道自己要什么。
- ❌当场对实现方案拍板（"那就用 XX 技术做"）——越界，把实现还给研发。
- ❌"我也不知道要什么，你看着办"——这才是真露怯。**宁可说"我回去把边界写清楚再来对"。**

## 5. 每个需求"做完长什么样"（你怎么知道收工了）

| 需求 | 完成的标志（验收） |
|------|--------------------|
| 1 条件清单 | 后台有一版干净、通用、能被模型解析的标准条件库；20~30 条定稿、过评审；成为上汽/赛力斯通用基线 |
| 2 对接意图 | 610 节点，用户说条件话术能被正确建成条件任务（抽样解析准） |
| 3 益红事件表 | 表里每个 `?` 都有准信；可做的进入配置 |
| 4 机器人 | 机器人事件清单入库、打标赛力斯，待统一规划 |
| 5 触发机制 | 赛力斯版触发引擎支持"状态变更"判断 + 多条件组合（任一/全部满足） |
| 6 协议拍平 | 赛力斯版协议支持任意位置、不吃组合层级（上汽版先用模型顶） |
| 7 定时器 | 支持"10分钟后/情人节10点"类时间条件，弱网隧道仍能触发 |
| 8 距离事件 | 支持"快到家时"（离指定点<N米）类地理条件，稳定触发 |
| 9 端云分工 | 简单/弱网任务端侧直接闭环不绕云；冲突任务在云上正确消解 |
