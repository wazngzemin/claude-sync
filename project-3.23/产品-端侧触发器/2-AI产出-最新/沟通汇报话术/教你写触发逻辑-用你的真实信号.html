教你写"触发逻辑" · 用你表里真实的 pass 信号

你已有：信号清单（表B「可订阅信号能力清单」，pass=实车测过可用）｜你缺：把信号写成"逻辑"（触发条件）

你手里的表（pass 那 61 条）= 有哪些信号能用，但没有"逻辑"。
所谓"逻辑" = 给每条信号配上 运算符 + 值 + 动作，让它变成一个"会触发的条件"。下面教你怎么配。

① "逻辑"就是这个公式（一条条件 = 一行）

[信号] [运算符] [值] → [触发什么动作]
例：车内温度 大于 26℃ → 提醒/开空调
你表里有"车内温度"这个信号(pass)，但"大于26→提醒"这后半截要你写——这就是"逻辑"。

② 运算符怎么选（你不会的核心，3 种）

信号是哪种能用的运算符怎么判断例（你表里的真实信号）
数值型大于/小于/等于/区间值是数字（温度/速度/距离/电量）车内温度 大于 26｜车速 大于 120
状态-静态等于（当前是什么）所有状态信号都支持空调开关 等于 on｜座椅加热 不等于 关闭
状态-动态变更为（刚变成什么）只有备注标"支持变更为"的能用车门 变更为 开｜座椅占用 变更为 已占用

⚠️ 你表里 pass 信号中，只有 3 个标了"支持变更为"：车门状态、车门锁状态、座位占座信息。其它状态信号只能用"等于"（看不了"刚变成"，只看"当前是"）。

③ 手把手：从你表里挑信号，写成逻辑（跟着做一遍）

例：想做"上车就打招呼"

  1. 想清楚触发时机："有人坐下" = 座椅从没人变成有人
  2. 去你表里找信号：座位占座信息 / 状态(states) → pass ✅、备注支持变更为 ✅
  3. 选运算符：要的是"刚坐下"(动态) → 用 变更为（这个信号正好支持）
  4. 填值：取值范围里是 已占用/未占用 → 值 = 已占用
  5. 配动作：→ 情感advisor 打招呼
  6. 写成逻辑：座位占座信息.states 变更为 已占用 → 打招呼 ✅ 一条逻辑就写好了

反例（教你避坑）：想做"接近目的地500m提醒"，去表里找"剩余距离"——你 pass 里目的地只有 地址+到达时间，没有剩余距离，所以这条现在写不了，得先让刘杨开放"剩余距离"信号。这就是为什么要先有信号清单(字典)才能写逻辑。

⑤ 回答你的问题："是不是把表格给他就行了？"

给谁给什么给表够不够
弋途（项4·实车信号可获取清单）把 pass 的 61 条筛出来、标清每条支持的运算符（等于/变更为），给他≈ 够（信号表筛一下就行，他要的是"能订哪些信号"）
触发器/需求1（高频条件清单）不是给信号表，是上面 ④ 那种写好逻辑(条件+动作)的新表不够，信号表 ≠ 逻辑，逻辑要你写

结论：给弋途看"能订哪些信号"——筛出 pass 给他 差不多就行；但需求1 的"逻辑/条件清单"——得你照 ④ 写出来，给表不等于给逻辑。
