一页结论 旧文档该删什么 VLM/VQA 全链路 你要做什么

一页结论

1. 端侧触发器 scope 收窄

固定、常驻、整车生命周期不变、在系统设置里能开关的功能，不要硬塞进端侧触发器。上下车安全、睡眠提醒、迎宾、部分车企已有 CDC 逻辑，优先归车企/底软闭环。

2. 触发器核心是事件，不是状态库

信号库/Context 存状态和 condition；触发器负责把状态变化加工成“事件”，再 push 给 Advisor。Advisor 不应该 24 小时轮询 Context 等事件。

3. VLM/VQA 要拆成两条链路

端侧 VLM 是持续观测和事件触发；云端 VQA 是一次性视觉问答。旧图把两者放在同一张大网里，容易讲混。

产品经理口径：触发器不是“所有自动化逻辑的容器”，它是把动态条件、生命周期任务和视觉事件标准化成可订阅事件的机制。端侧负责近场、低延迟、弱网可用；云端负责注册、语义拆解、事件组装、全局仲裁和 Advisor/Director 推理。

VLM 与 VQA 的完整链路：必须拆成两条

A. 端侧 VLM 持续事件链路（赛力斯/AIBox 能力主线）

适用场景：“帮我照看二排宝宝，被子掉了提醒我”“用户睡醒时告诉我”“驶出地库时提醒我”。它的本质不是问一次图，而是在任务生命周期内持续观测某类视觉状态变化。
#环节产品上要讲清楚什么当前缺口
1用户 query / Director识别为持续视觉任务，创建动态 Advisor 或条件任务。query case 清单不完整。
2动态 Advisor 拆解把“看什么、多久看、触发后通知谁”拆成注意力 SP / 视觉事件订阅。SP 输出格式需和触发器注册 API 对齐。
3⑮云端触发器注册校验能力、绑定生命周期、生成 rule_id，判断是否需要下发端侧。通用视觉事件类型、lifecycle、cancel/replace 字段需定义。
4⑮→端侧/AIBox下发的是“增量观察任务”，不是覆盖 always-on 原 prompt。舱内静态/舱内动态/舱外三类任务路由需定义。
5AIBox 调度 + 端侧 VLMAIBox 排优先级、抢占摄像头、执行观测；触发器不做 AIBox 内部调度。谁接收云端下发任务并转给 AIBox，要单独对齐。
6④端侧信号库VLM 输出先写成客观状态，例如“毯子在地上”“宝宝闭眼”“副驾在打电话”。动态观测内容的字段/schema 需新增。
7⑮云端触发器组装事件状态变化被加工成“用户睡醒”“被子掉了”“驶出地库”这类事件。事件组装在云端还是端侧，会议倾向云端，但需拍板。
8⑰Advisor / ⑳DirectorAdvisor 消费事件并结合 Context 产出建议；Director 决定说什么、做什么。事件 push 协议与 Advisor 订阅协议未闭环。

B. 云端 VQA 一次性视觉问答链路（问一次答一次）

适用场景：“现在后排宝宝怎么样？”“前面是不是有加油站？”“拍一帧看一下”。它适合一次性问答，不适合高频持续盯守，也不能替代端侧 VLM 的事件链路。
#环节链路注意
1Director/Advisor 发起生成视觉问题 prompt。不是端侧触发器任务。
2视频通话/抓帧接口端侧摄像头抓一帧上传。会和 VQA/其他视觉业务抢摄像头。
3云端 VQA 模型图 + prompt → {result, description}。延迟和网络依赖较高。
4Context 注入结果作为事实写入 Context。字段名、格式、写入方待确认。
5Advisor/Director 消费根据 VQA 结果回答用户或产生建议。不适合“每 3 秒盯一次”的场景。
最重要的产品判断：如果用户要“持续盯着某个变化”，优先走端侧 VLM 事件链路；如果用户只是“现在帮我看一眼”，走云端 VQA。旧文档把“定时器 + VQA”和“端侧 VLM task”并列但没有明确边界，这是需要删繁就简的地方。

端侧触发器、云端触发器、VLM/AIBox 各自做什么

模块应该做不应该做本次会议后的定位
⑮ 云端触发器 注册校验、规则入库、端云分类、生命周期管理、事件组装、push Advisor、全局/语义仲裁、规则包编译下发。 不直接替代 Director 做最终响应；不把所有状态都当事件推。 本次主控中枢。视觉事件最好由它统一组装，Advisor 订阅它的事件。
㉔ 端侧触发器 接收已校验规则包、本地持久化、本地定时器、读取端侧信号库、弱网下执行已下发且白名单内的动作、结果回写。 不吃自然语言；不做复杂语义判断；不接管车企固定系统功能；不做 AIBox 内部调度。 从“端侧万能闭环引擎”收窄为“云端下发任务的本地履约层 + 视觉事件通道”。
AIBox/VLM 执行 always-on 和视觉观测任务，输出客观视觉状态；做摄像头资源和模型请求优先级排序。 不直接决定业务怎么响应；不替代 Advisor/Director。 VLM 是眼睛，AIBox 调度是视觉资源管理者，触发器只是任务/事件通道。
Context 存状态、condition、事实快照、VQA/VLM 结果，供推理读取。 不要被讲成事件源；Advisor 不应轮询它等事件。 事实源，不是事件分发器。
