项5 · 视觉类长时任务 兜底拒绝策略 P0·630 · 依赖感知 · 火山
📋 内容（拒绝/调度策略）
规则内容
最大并发同时最多几个长时视觉任务
互斥组VLM_CABIN(舱内)/VLM_FRONT(舱外) 不能同占一路
终态success/empty/timeout/failed（空≠没看；超时不写假结果）
兜底拒绝超并发/超时 → 明确返回拒绝，别假装在看
🎬 case 跑逻辑（手把手："照看后排宝宝"看不成时，为什么必须诚实拒绝而不能装看）
🅰 配置期：长时视觉任务的调度规则先定死
  1. 弋途advisor 创建一个长时视觉任务"持续照看后排宝宝" → 提交给感知器 { vtask_id:"V-9", target:"后排", lens:"VLM_CABIN", priority:"P2", mode:"持续" }
  2. 你/感知器按规则把它排进队，检查三条：①最大并发没超 ②VLM_CABIN 这一路没被占 ③优先级够不够 💡为什么要规则：摄像头/VLM 算力有限，不能无限开长时任务，否则全卡死、互相抢路
🅱 运行期：关键在"看不成"那一刻怎么办
  1. 感知器正常路径：调 VLM_CABIN 看后排 → 返回 success: 宝宝睡着了 → 推给 advisor
  2. 感知器冲突路径：此刻来了更高优 P0 任务（如急刹后查驾驶员状态）抢占了 VLM_CABIN → 你的宝宝任务排不进 / 超时
  3. 你不能假装在看，必须回明确终态：timeout(超时) 或 rejected(被抢占) → 告诉 advisor"这次没看成" ⚠️ 为什么不能回空 empty 当"没异常"：advisor 会以为"宝宝没事"，可你根本没看 → 漏看 = 安全事故。所以铁律是「空≠没看」「超时绝不写假结果」
  4. 弋途advisor 收到拒绝 → 降级处理：3 秒后重试 / 或语音问"后排宝宝还好吗？"
