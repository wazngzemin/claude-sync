# Knowledge Selector 延迟优化方案

## 现状诊断

| 指标 | 当前值 | 目标 |
|------|--------|------|
| Selector 延迟 | 1,805ms | < 500ms |
| Selector tokens | 3,355 | < 800 |
| 总链路（Selector + Director） | ~6.5s | < 4s |

**瓶颈拆解**：
- Input tokens 占比 ~95%（3355 中大部分是 system prompt 里的 13 个场景描述）
- Output tokens 极少（只输出一个 ID）
- 但 TTFT（Time To First Token）受 input 长度直接影响

## 优化层级（由快到慢实施）

### Level 1：压缩输出格式（省 ~100ms）

当前输出要求 reasoning，改为纯 ID 输出。

```
// Before
{"add_knowledge_ids": ["DRV-01-01-05"], "remove_knowledge_ids": [], "reasoning": {"add_reason": "...", "remove_reason": "..."}}

// After — 纯数字
012
// 或多命中
001,013
// 无命中
000
```

效果：output tokens 从 ~50 降到 ~5，省约 100-200ms。
代价：失去可调试的 reasoning 字段。折中方案：仅在 debug 模式输出 reasoning。

### Level 2：压缩场景索引（省 ~400-600ms）★ 核心优化

当前每个场景描述 200-400 字，13 个场景 = ~4000 字 system prompt。
模型做的其实是**语义分类**，不需要读完整描述。

**压缩策略：关键词锚定 + 互斥规则分离**

```
## 场景索引（仅匹配用）
ID_001 儿童陪伴安抚 | 锚词：照顾孩子/哄孩子/陪孩子玩/放动画/儿歌/故事/益智/安抚情绪/哄睡/猜谜/逗孩子
ID_002 后排观影 | 锚词：后排看/吸顶屏/看电影/追剧/综艺/纪录片

ID_006 副驾座椅 | 锚词：副驾座椅/女王副驾/副驾调节/副驾不舒服

ID_008 冥想空间 | 锚词：冥想/压力大/正念/午间放松/情绪舒缓+驻车
ID_009 车内小憩 | 锚词：打盹/午休/小憩/休息一会/叫醒/睡一下

## 互斥规则（仅当多场景疑似命中时参考）
001 vs 013：「孩子睡了别打扰」→013 | 「陪孩子玩」→001
001 vs 009：「孩子困了要睡」→009 | 「哄孩子开心」→001
003 vs 002：驻车→003 | 行车后排→002

```

效果估算：
- 场景索引从 ~4000 字压缩到 ~600 字
- Input tokens 从 ~3000 降到 ~800
- TTFT 预计降低 40-60%（约省 700-1000ms）

### Level 3：两阶段检索（省 ~300ms 额外）

如果场景数量未来扩展到 50+，纯 prompt 内分类会持续膨胀。

**方案：embedding 预筛 + LLM 精排**

```
Stage 1: Embedding 相似度检索（< 50ms）
  - 将 13 个场景的锚词做 embedding
  - 用户 query embedding → top-3 候选场景
  
Stage 2: LLM 精排（< 300ms）
  - 只把 top-3 场景的压缩描述送入 LLM
  - Input tokens 从 800 降到 ~300
  - LLM 只做 3 选 1 的精排 + 互斥判断
```

效果：即使场景扩展到 100+，延迟恒定在 ~350ms。

### Level 4：规则前置 + LLM 兜底

对高频、高确定性场景用正则/关键词规则直接命中，跳过 LLM。

```python
# 高置信度规则（覆盖 ~40% 请求）
FAST_RULES = {
    r'(抽烟|通风|换气|车里味)': 'ID_012',
    r'(冥想|正念)': 'ID_008',
    r'(打盹|午休|小憩|睡一会)': 'ID_009',
    r'(5[dD]|沉浸.*影院)': 'ID_007',
    r'(雨刮|下[雨雪]|起雾|除雾)': 'ID_010',
    r'(宠物|狗|猫).*(?:留|热|叫|车)': 'ID_011',
}
# 未命中规则 → 走 LLM 分类
```

效果：高频场景 < 10ms，整体平均延迟降 30-40%。

## 实施优先级

| 优先级 | 措施 | 预期收益 | 实施成本 |
|--------|------|----------|----------|
| P0 | 压缩场景索引（Level 2） | -700ms | 低：改 prompt |
| P0 | 精简输出格式（Level 1） | -150ms | 低：改 prompt |
| P1 | 规则前置（Level 4） | -300ms avg | 中：加前置逻辑 |
| P2 | embedding 预筛（Level 3） | 可扩展性 | 高：加向量服务 |
