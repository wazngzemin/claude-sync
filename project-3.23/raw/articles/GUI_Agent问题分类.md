# GUI Agent 问题分类

## 截止日期
2026-03-30

## 问题分类

### 1. 指令理解错误
**问题描述：** 把澄清指令当成闲聊

**解决方向：**
- 补充知识，明确区分闲聊和任务处理
- 通过示例教会模型识别澄清类指令
- 通过指示强化任务处理意识

### 2. 工具参数混淆
**问题描述：** planner需要输出两种参数（创建任务/补充信息），经常弄混

**根本原因：** 下游工具难度高

**解决方向：**
- 简化planner输出，只输出自然语言描述
- 单独拆一个agent到下游处理参数区分
- 用Flash模型做二次思考，判断是补充信息还是新任务
- 将GUI相关知识注入Flash模型

### 3. 任务难度高
**背景：** GUI任务本身复杂度较高

**影响：** 导致各类问题频发

## 优化策略

### 短期方案
1. 补充知识和示例，解决指令理解问题
2. 工具链路重构，简化planner职责

### 中期方案
1. 工程分解：用小模型处理下游工具细节
2. 减轻主director压力
3. 所有复杂工具（如生图工具）都考虑前置模型思考

## 行动建议
- 体验Pad实际使用场景，感受问题
- 参考badcase清单逐个分析
- 多讨论解决思路
