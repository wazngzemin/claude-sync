给王泽民改文档(讲稿/PRD/图)时,**不要擅自删内容、去重、重构结构**。他要"优化话术/风格"时,默认含义是"在原内容上打磨措辞",**不是**删段落、不是改结构。

**Why**:本会话连犯两次——他要 renwei 优化讲稿话术,我却先"整篇去重"把他第一~十二段讲稿正文删了,他暴怒"你全给我删了前面的"。后又全量还原成 4 个重复版,他又嫌"全是重复"。两头不讨好,根源都是**替他做了删/重构的决定**。

**How to apply**:
1. 改文档先**只做加法/改词**,不删段、不动结构。要删/合并/去重,**先问**(用 AskUserQuestion 给他拍板留哪版),别猜。
2. **每次大改前先 `cp` 一份带后缀备份**(`_X前备份.html`),留兜底,改完告诉他备份在哪。
3. renwei 话术优化要忠于"少动":他的讲稿本就是好口吻(带例子/自问自答),AI 一磨人味就没——只清真 AI 痕迹(破折号/三段对仗/金句/拔高),其余保留,逐处交代。
4. 他认可的方案要落到**唯一交付物**,别留一堆并行版本。

讲稿现状(2026-06-15):`产品-端侧触发器/触发器流程图-评审讲解稿.html` 已合并为单一版=大图+开场+主持人分段第一~十二段(内嵌15/24原子·VLM精讲)+谁写代码+结语。**每段有两个图都要留**:① mini-flow 文字摘要条(用户明确"还是需要文字的",别删);② 该段独立流程图——**只含该段模块、重新连线重画**,不是大图截框(`diagrams-v2/gen_segflow.py`,segflow01~12.svg)。教训:他要"每段流程图"=重画干净子图,截大图局部框他嫌乱。详见 [[project_trigger_architecture]] [[feedback_trigger_diagram]]。
