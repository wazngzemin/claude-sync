**导航类触发器事件（接近/到达/离开）PRD**，王泽民负责，P0·630量产。源素材=《触发器-导航信号需求-给刘杨.md》§3~§8（信号编号119~133）。

**核心难点（决策二·待张航/刘杨拍）：** "接近"≠"距离<500"。用户从500m开到公司，499..50m每帧都<500会刷屏几十次。要的是"第一次跨过500m提醒一次就闭嘴"=**one_shot**，靠**POI状态机**（远→已接近→已到达→已离开，翻格自锁）实现，现协议无记忆做不了。两条路：**方案A**=状态机放触发器引擎(统一可复用、改引擎工期没底)；**方案B**=端上算好"接近事件"上报(快、散、多端各写)。**产品倾向A，听刘杨工期；A顶不上630则B兜本期、A下版。沟通刘杨的关键=逼出"A的工期"这一个数。** 死区480/520防抖；距离用沿路线非直线；阈值按场景产品配。

**非目标纠正（重要）：** "纯兜风/隔江景点做不了"措辞太绝对——电子围栏(需10/11)也是触发器的活、能做预置区域；只有"完全无目标即兴兜风"才真做不了(转Director持续任务/VLM丁斌)。别把围栏甩给益图。

**音频类触发信号：** 张航5/31赛力斯会已拍归触发器("音乐播放状态都要放进去")。补音量/切歌/循环/播放状态信号，支持"循环切歌超10次→关怀"。清单已给测试，待实测支持哪些。老板问过、要主动报。
