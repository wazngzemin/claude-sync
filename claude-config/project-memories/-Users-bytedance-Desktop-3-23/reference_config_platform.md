火山引擎 AI 汽车管理平台「触发器 → 场景管理 → 新建场景」= 架构图节点 13(规则配置平台)的真实后台。2026-06-15 用户截图确认,据此把节点13拆成 8 原子。

**真实页面分区(= 13-1~13-8):**
- 基础信息:场景名称(【测试/正式】-【使用方】-【场景】)、优先级(1)
- 发布范围:渠道(上汽DNA)/指定车辆VIN(多选)/适用车型/软件版本(运算符) = effective_scope
- 触发条件:**动态条件**(vlm信号·选信号/字段/运算符/值·多组·且/或)+ **静态条件**(车状态信号·24类字段·多组)
- 执行动作:舒适/情感/出行/内容 Advisor + **主动推荐** + **延时执行** + 事件/注意力信息 + 下游输入(prompt)
- 执行策略:执行总次数/单次上电周期/自然日内总次数/执行最小间隔(默认30秒) = execute_policy
- 「新建场景」按钮 = 注册

**车状态信号 24 类字段**:车辆行进状态/车内空调/空调循环/空气净化/座椅按摩/方向盘加热/音量/当前播放音频/近光灯/远光灯/后雾灯/轮胎/车内外温度/车门/座位占座/能量回收/座椅通风/座椅加热/导航/氛围灯/车辆速度/车窗/车门锁/行驶里程。

**执行动作下拉**:舒适Advisor/情感Advisor/出行Advisor/内容Advisor/主动推荐/延时执行。

**⚠ 需改造点(已标进节点13原子表)**:① 动态条件现在只有"vlm信号"占位,要补 VLM 7字段(camera_group/seat/query/interval/timeout/priority/mutex_group);② 运算符要补 range/持续时长/group与或/任一车门·座位;③ 执行动作要补端侧直执白名单(TTS/卡片/车控)+ custom_callback;④ 要补端云分类(哪些下发端侧24/留云)+ 端侧本地倒计时下发(弱网)+ 撤销/灰度(update_status 1生效2失效3删)。
