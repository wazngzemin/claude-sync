# CQC Bug 分拣带教 · 逐字话术稿

> 使用说明：本稿可基本照念。中括号 `[ ]` 内是给带教人（王泽民）的动作提示，不用念出来。
> 全程约 2–2.5 小时讲完，配合实战。建议打印「决策树」「工具速查表」贴在 CQC 工位。

---

## 开场（3 分钟）

> "来，咱们花两天把你带出来。说清楚目标：坤姐定了，你以后负责 **bug 分拣**——一个 bug 进来，你判断它该谁修，顺手写下你认为正确的处理思路。
>
> 先给你卸个包袱：**你不用改 SP，不用上线示例，也不用去 Fornax 验证——因为权限还没下来。** 你只干两件事：**第一，把 bug 送到对的人手里；第二，写出你觉得它该怎么处理。**
>
> 你肯定会想'我没法验证，万一分错了咋办'。记坤姐一句话——**'改不了、验不了，不等于分发不了'**。你分错了，下游会转回来，我再带你纠。所以咱们的原则是：**别怕分错，但要尽量分对、分快。** 你存在的意义，是别让两三百个 bug 全堵在明骏一个人那儿。听明白了吗？"

[等他回应，确认他理解"自己是分拣岗，不是修复岗"]

## 第一节：先搞懂 Planner 在干嘛（15 分钟）

> "你要想分对 bug，得先知道这个系统是怎么工作的。我画给你看。
>
> [画图或指屏] 用户说一句话 → 转成文字 → 进 **Planner 这个大脑**：它要听懂你想干嘛，然后从 **23 个工具**里挑一个，写好命令 → 交给下游的车控、导航、音乐去执行。
>
> 你记住一句话：**你看的 bug，百分之八九十都错在 Planner 选工具这一步——本该调 A 工具，它没调，或者调成了 B。**
>
> 我考你一个：'打开冰箱电源'这个 bug 失败了，你觉得最可能错在哪？"
>
> [等他答。正确答案引导：] "对，Planner 没认出'冰箱电源'是个车控指令，没去调那个车控工具。这就是最典型的一类。"

> "再给你一套口诀，抄本子上——遇到 bug 先按这个顺序问：
> **第一，这请求到没到 Planner？**（没到就是链路崩了，不归咱们）
> **第二，工具选对没？**（选错就是 SP 或动态示例的事）
> **第三，信息够不够？**（缺记忆、缺车况就是上下文问题）
> **第四，信息都齐了它还错，那才是模型笨。**
> 口诀：'先问到没到，再问选对没，最后才怪它笨。'"

## 第二节：五个出口和决策树（25 分钟）—— 核心

> "好，现在讲最重要的。你分拣，本质就是把每个 bug 丢进**五个桶**之一。我一个个说：
>
> **桶一：不是问题。** query 本身就是乱的，或者系统压根没这功能。比如'打开 MV 跳转短视频工具'——咱们根本没这工具。这种你**直接评论'无此功能/query 构造有误'，关掉**，不用往下走。
>
> **桶二：链路问题。** query 挺正常，但请求根本没到 Planner，是客户端或工程崩了。这种**转工程**。
>
> **桶三：静态示例，也叫 SP。** 这是一**类**能用一条规则覆盖的问题。比如所有'下周末是几号''元旦是哪天'算错了——这是一类日期问题，改一个地方教模型'日期一律查 timestamp'就全解了。这种**转单如水**。
>
> **桶四：动态示例。** 这是你最常遇到的，占八成。它是**低频、单点、就事论事**的——一句话能说清调哪个工具。比如'动力响应设置为舒适'没识别成车控，你写一句'调用车控工具下发即可'就行。这种**写好草稿给李强**。
>
> **桶五：动态知识。** 模型缺的是某个事实、某个车的规格。比如它不知道这车有没有'吹腰模式'。这种**转算法**补知识。
>
> 还有个特殊标记——**疑似下游**：你判断 Planner 是对的，但怀疑下游车控根本不支持（比如'打开加油小门'）。这种你照写草稿，但**额外标'疑似下游'，进左晨的群确认**。"

[拿出打印的决策树贴他工位]

> "这张决策树贴你工位。每个 bug 你就从上往下走：**先看 query 乱不乱 → 再看到没到 Planner → 再看是一类共性还是孤立单点 → 最后看要不要标下游。** 走顺了，五秒钟就能定位一个 bug。"

### 决策树（打印件）

```
【一个 bug 进来】
│
├① query 本身就乱 / 系统没这功能?
│   例:"打开MV跳转短视频工具"(没这工具)
│   → 出口【非问题】:评论"无此功能/query构造有误",关闭
│
├② query 合理,但根本没走到 Planner(链路/客户端崩)?
│   → 出口【链路问题】:转工程
│
├③ 走到 Planner 但工具调错/没调 → 问:单点 还是 一类共性?
│   ├A 一组能用一条规则覆盖的(所有日期计算 / 所有json格式)
│   │   → 出口【静态示例/SP】:转单如水
│   ├B 低频、明确、就事论事的单点(占工作量 80%)
│   │   → 出口【动态示例】:写决策分析草稿 → 给李强  ★主战场
│   └C 模型缺的是"知识/事实"(车辆规格盲区)
│       → 出口【动态知识】:转算法
│
└④ 看着能写动态示例,但怀疑下游车控不支持?
    例:"打开加油小门""切换声场模式"
    → 照写草稿,但额外【标"疑似下游"】→ 进左晨群确认
```

## 第三节：工具路由速查（20 分钟）

> "判到'该走哪个工具'的时候，你不用背，对着这张表查。我讲几个最高频的：
>
> **vehicle_basic_control——车控总入口，十个 bug 里四个是它。** 车窗、空调、座椅、车灯、雨刮、天窗、门锁、充电加油盖、冰箱，还有**驾驶设置**：动力响应、转向手感、能量回收、动力源，这些冷门词模型经常认不出来。决策分析就一句：'无需改写，调用 vehicle_basic_control 工具下发命令即可。'
>
> **vehicle_system_settings——系统类：** 热点密码、声场、音效、音量、屏幕亮度、蓝牙、打开某个 APP。
>
> **vehicle_communication_control——通讯：** 打电话、挂断、同步通讯录。注意发短信不支持，要澄清。
>
> **goal_list_update——这个你要特别记：凡是带条件、带时间、要持续做的，都走它，不是车控！** 比如'30 秒后开车窗''温度低于 22 度关空调''一路给我介绍风景'。
>
> 剩下的单点工具一看就知道：查天气 search_weather、查车况 search_vehicle_status_info、查导航 POI 是 search_poi_qa、看车里几个人是 search_visual_info、注册人脸 face_id_register。"

## 第四节：三个最容易错的坑（20 分钟）—— 重点演练

> "我专门挑三个你百分百会栽的坑，先讲再练。
>
> **坑一：带条件/带时间的，要走 goal_list，不是车控。**
> '直接打开车窗'是车控；但'30 秒后打开车窗'就是 goal_list，因为它有'30 秒后'这个触发条件。**口诀：看到'…后''…时''持续''每次'，立刻往 goal_list 想。**
>
> **坑二：music 和 media 别搞混。**
> 要'找一首歌、放周杰伦'——那是要搜内容，走 music；'暂停、关 MV、视频暂停'——那是控制正在放的，走 media。**口诀：找歌用 music，控当前用 media。**
>
> **坑三：声场、音效、音量这种，边界本来就模糊。** 我跟你说实话——连原始表里都不一致，'切换声场'标了系统设置，'场景音效'标了车控。所以**这类你拿不准就标'边界存疑'，别硬判，交上来一起定。** 这不是你水平问题，是这块本来就需要人一起确认。"

[现场出 3-5 个 query 让他口答，每错一次回扣口诀]

## 第五节：怎么写决策分析（15 分钟）

> "判到动态示例桶的，你要写一句决策分析。格式很简单：
> **query：照抄原文，不许改。**
> **决策分析：要不要改写 + 调哪个具体工具名 + 干什么。**
>
> 三条死规矩：
> **一，必须写具体工具名**，不能写'把空调打开'，要写'调用 vehicle_basic_control'。
> **二，一句话能说清就别展开。** 如果你发现要写好几个'如果…就…'，那是信号——**这不该是动态示例，该交静态，停手上交。**
> **三，称呼别改。** '给妈妈打电话'就写妈妈，别改成号码。
>
> 我给你看个真实的反面教材：表里有一条'打开录音开关'，决策分析却写成了'结束录音纪要'——**打开写成了结束，这就是复制粘贴串行错了。** 你写完一定要自查一遍：**动作和 query 对得上吗？** 这个自查能力，比写得快更重要。"

### 决策分析模板与正反例

```
模板：
query：<原文，不改写>
决策分析：<是否改写 + 调哪个具体tool_name + 干什么>
```

| ✅ 好 | ❌ 坏 |
|------|------|
| 无需改写，调用 vehicle_basic_control 工具下发命令即可 | 让模型把空调打开（没说调哪个工具）|
| 调用 search_vehicle_status_info 查氛围灯当前颜色 | 看一下氛围灯（没点名工具）|
| 调用 goal_list_update 创建"温度低于22度关空调"的条件任务 | 温度低就关空调（没说是条件任务/工具）|

## 第六节：上真实数据实战（40 分钟）—— 用「解 bug」评测表

> "光讲没用，咱们上真家伙。这是评测表，你只看一列——**模型输出评分**。**0 分、1 分、2 分的，都要你来分发；3 分是合格的，不用管。** 这次一共 44 个。
>
> 而且表里已经有一列'问题分类'帮你做了一半——**你先看这列，再走决策树，能快一倍。** 我教你怎么对应：
> - 它写'工具调用类'、又是孤立单点的 → 多半动态示例
> - 它写'推理理解'里的日期算错、json 格式 → 这是一类，走静态
> - 它写'上下文'——结合车况、记忆没做好 → 走静态通则
> - 它写'硬件认知缺失' → 走动态知识
> - 它写'平台&下游链路' → 转工程或左晨
>
> 来，我先演示三个，你看我怎么想。"

[演示，边做边出声思考：]

> "[第 1 个] 你看这 8 行——'认识一下副驾的明骏''认识王迪''记住他叫小华'，全是 json 格式问题。**重点来了：同一个格式 bug 出现了 8 次。你会一个个写 8 个动态示例吗？不会——这就是该改 1 处 SP 的铁证，整组扔静态桶，转单如水。** 这是你今天最该记住的一课：**成批同因 = 静态，孤立单点 = 动态。**
>
> [第 2 个] '下周末规划个杭州两日游'——日期算错了。这跟'元旦''北京周边'是同一类，全是日期算错。也是一类，走静态，教模型查 timestamp。
>
> [第 3 个] '驾驶风格激进一点'——它没调车控的运动模式。这个是孤立单点，一句话能说清：'调用 vehicle_basic_control 调为运动模式'。这个才是动态示例，写草稿给李强。"

> "看懂套路了吗？现在你来。我遮住答案，你分这 20 个，每个告诉我：**进哪个桶 + 调哪个工具 + 要不要标下游。** 错了我当场跟你说错哪。"

[让他做，你按"出口对不对/工具对不对/风险漏没漏/草稿质量"四条判。纠错话术见下]

**纠错话术示范**（别只说"错了"）：

> "这个'我比较累时给我放首歌'你判成了音乐工具——再读一遍，'比较累的时候'是个条件，这是 goal_list_update。回扣口诀：'…时'就往 goal_list 想。"

### 实战参考答案（带教人留底，不发给 CQC）

**一类共性 → 转单如水（静态示例）：**
- 日期算错：元旦哈尔滨、下周末杭州、北京周边、第二天加古北水镇、景点顺序换、删下午茶（共 6 条，同因）
- json 格式：认识副驾明骏/王迪、记住叫小华、看持仓股票、关音乐再认识小璐、认识左后小明（共 8 条，同因）
- 行车安全/端状态：行车中找短视频、听歌、看剧（共 4 条）
- 记忆未结合：上次那家店、讨厌鬼、上次追的剧（共 3 条）
- 端状态：看宝宝睡没调风量
- 工具混用：删海底捞记忆(personal_skill与记忆混)、改唱燃烧爱火
- goal_id 格式：删座椅加热任务（2 条）
- 话术：导航啰嗦、大冒险主体错
- 多工具并行：关音乐再认识妹妹

**孤立单点 → 给李强（动态示例）：**
| query | 决策分析草稿 |
|-------|------------|
| 我比较累时给我放首歌 | 调用 goal_list_update（条件触发型）|
| 删掉我喜欢吃海底捞的记忆 | 调用 operate_user_memory |
| 驾驶风格激进一点 | 调用 vehicle_basic_control 调为运动模式 |
| 跟车远一点 | 调用 auto_drive，跟车距离调高一档（≠最高5档）|
| 氛围灯现在啥颜色 | 调用 search_vehicle_status_info 查状态（不是控制）|
| 靠边停车 | 调用 auto_drive（一键贴边）|
| 编排专属通勤节目单 | query 改写后调用对应工具 |
| advisor：小林搞怪 | 调用声音演绎/ai_broadcast |
| 告诉他还有15分钟到 | 不支持，话术向用户解释 |
| 冥想十分钟 | 结合"我们"开副驾座椅按摩（边缘，可标存疑）|

**硬件认知缺失 → 转算法（动态知识）：**
- 我的腰好酸（无吹腰模式）、椅子调舒适些（无舒适模式）、打开儿童锁（不支持，话术告知物理开关）、后背黏黏（无吹背模式）

**难解 → 算法深查：**
- 第三天景点顺序换下（行程多轮内容幻觉）

## 第七节：分完往哪放（10 分钟）

> "分完每个 bug，你要在表里填四样，统一格式李强他们才好接：
> **一，出口**：动态示例 / 静态示例 / 下游 / 非问题 / 链路。
> **二，决策分析**：只有动态示例桶才填，按模板一句话。
> **三，风险标记**：疑似下游 / 边界存疑 / 复杂建议静态。
> **四，处理人**：李强 / 单如水 / 左晨 / 工程。
>
> 分流规则记牢：**动态示例给李强，下游进左晨群，静态给单如水，非问题你自己评论关掉。**
>
> 还有个习惯——**先去重。** 这 44 个里'认识副驾明骏'有 3 份、'删海底捞记忆'2 份，同一个 query 你合并成一个处理，别重复干活。"

| 风险漏标 | 疑似下游/非问题没标 | **零容忍** |
| 草稿质量 | 没点名工具/动作和 query 不符 | 一般 |
