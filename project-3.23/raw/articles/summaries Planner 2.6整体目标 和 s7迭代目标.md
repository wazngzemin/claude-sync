[summaries] Planner 2.6整体目标 和 s7迭代目标
概述
本文档定义了Planner 2.6的整体性能目标、细化体验目标以及S7迭代的具体目标，涵盖复杂车控、任务拆解、对话融合、主动服务、任务和交互管理等方面。
关键章节
1. 车场景痛点
复杂多步任务需要用户一直给输入，模型缺失上下文（口味、冷热、价格偏好等）
用户只有大目标没有小目标，不知道功能怎么用
检索想要的东西很费劲
一锤子买卖，需求不闭环
闲聊聊不下去，割裂没内涵
2. 2.6整体目标
性能目标
首字响应P50 <= 500ms
首字响应P90 <= 800ms
推理ACC >= 90%
回复ACC >= 93%
细化体验目标
复杂车控【s7】：ACC >= 90%，控制完成P90 <= 2000ms
任务拆解：结合context（情景、状态、记忆）进行思考
对话【s7】：完成对话融合，planner直接回复普通闲聊
任务和交互管理【s7方案，s8-s10上线】：任务打断恢复、多任务冲突管理
界面【s6-s7工程版】：开发工程版界面
3. S7迭代目标
S7性能目标
planner首token P90 <= 1200ms
推理ACC >= 90%，回复ACC >= 93%
S7细化体验目标
不加意图router模块
做好单步任务的拆解和评测（车控、导航、媒体）
对话融合：planner结合context完成对话任务（准确率>=93%）
任务拆解结合context
打通主动服务：执行类简单介绍理由+询问或直接执行；情感类直接讲
仲裁意图
合并TOB线简单复杂和意图
S7输出产品和技术方案
S8完成新模型输出和主链路接入
S9-11持续迭代模型，目标ACC>=96%，P50<=150ms，P90<=250ms
核心要求
S7完成ACC、耗时准确摸底
不加意图router，简化链路
单步任务拆解重点跟踪车控、导航、媒体
主动服务需区分执行类和情感类
仲裁意图模型需重新训练
提取的实体
Planner 2.6：任务规划模块的2.6版本
S7：第七个迭代周期
主动服务：系统主动发起的服务和建议
仲裁意图：合并简单复杂仲裁和意图判断的模块
提取的概念
首字响应：从用户说完到系统开始回复的延迟
对话融合：将对话能力整合到Planner中的架构调整
主动服务：基于场景和状态主动为用户提供帮助的机制
