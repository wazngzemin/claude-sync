# 会议复盘助手

这个 skill 帮助用户对会议进行完整复盘，包括提取会议要点、整理待办事项、生成结构化的会议记录。

## 使用场景

- 会议结束后需要整理会议内容
- 需要从会议记录中提取关键信息
- 需要生成待办事项清单
- 需要对会议进行复盘总结

## 完整复盘流程

### 第一步：获取会议原始信息

请用户提供以下会议信息之一：
- 会议录音/转录文本
- 会议纪要草稿
- 会议要点笔记
- 聊天记录或邮件

### 第二步：提取会议关键要素

从提供的信息中提取：
1. **会议基本信息**
   - 会议主题
   - 会议时间
   - 参与人员
   - 会议地点/形式

2. **讨论内容**
   - 议题一：讨论内容、结论
   - 议题二：讨论内容、结论
   - 议题三：讨论内容、结论
   - （根据实际议题数量调整）

3. **决策事项**
   - 已确定的决策
   - 待进一步讨论的问题
   - 需要确认的事项

### 第三步：整理待办事项

识别并整理所有待办任务：
- **待办事项1**：具体内容 | 负责人 | 完成时间
- **待办事项2**：具体内容 | 负责人 | 完成时间
- ...

确保每项待办都有明确的负责人和截止时间。

### 第四步：生成结构化输出

生成以下格式的会议复盘文档：

```
# 会议复盘

## 基本信息
- 会议主题：
- 会议时间：
- 参与人员：
- 会议形式：

## 会议要点

### 议题一：[名称]
- 讨论内容：
- 结论：

### 议题二：[名称]
- 讨论内容：
- 结论：

## 决策事项
1. 
2. 

## 待办事项
| 序号 | 待办内容 | 负责人 | 完成时间 |
|------|----------|--------|----------|
| 1    |          |        |          |
| 2    |          |        |          |

## 下次会议安排
- 时间：
- 议题：
```

### 第五步：复盘建议（可选）

根据会议内容，可以提供以下维度的复盘建议：
- 本次会议的目标达成情况
- 做得好的地方
- 需要改进的地方
- 下次会议需要跟进的事项

## 输出要求

- 使用中文输出
- 内容完整、结构清晰
- 待办事项必须有明确的责任人和时间
- 保持客观、中立的语气

## 注意事项

- 如果用户提供的信息不完整，先询问补充
- 如果无法确定某项待办的负责人，标注为"待确认"
- 如果有紧急事项需要特别标注
