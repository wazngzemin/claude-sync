# 文章吸收解析协议（absorption-analysis）

泽民日常丢文章（自我成长/职场/技术/架构/产品方法论）的**默认产物**。不是 fork 三件套、不是晨读判级、不是梁宁复盘——那三样只在他当轮明确要求时才走。本协议是 2026-07-17 一篇文章连改 9 版（v1 fork 工具→v2 复盘→v4 抽象化全被打回）换来的定版，逐条都是他亲口纠正过的。

## 触发

用户丢文章/链接/原文，说"解析 / 帮我吸收 / 讲透 / 对我有什么用"。

## 第 0 步：分诊（开头必给，先于一切）

页面最顶部给「分诊结论」块：

- 文章给的是**一副看问题的透镜 / 一套认知** → 做解析文档
- 文章给的是**一个会被反复填、反复算、反复判的流程**（评分卡/决策矩阵/可勾选诊断）→ 做工具（fork 管线的 tool.html）
- 拿不准 → 先解析，工具当内嵌小件后补

## 产物形态（定死）

一个自包含 HTML，存：


2. **qiaomu-design-advisor（设计）**：视觉方向**已定死 = 暖纸亮色·内容优先**（他 2026-07-17 嫌暗后定的），不再 AskUserQuestion 问方向。

## 视觉系统（已定版，直接抄样张）

- 亮色暖纸：`--bg:#f4efe4`，深字 `#2c2620`，琥珀重音 `#b07a1f`，赤褐 `#a85a34`，青 `#2f7d63`，土红 `#bb5238`
- 正文 16px / 行高 1.8 / 阅读列 ~760px；衬线大标题（Songti SC）+ 无衬线正文
- 左侧 sticky 目录 + 顶部进度条 + 右上角亮/暗切换（`html[data-theme="dark"]` 变量组）
- SVG 图一律用 CSS 变量类（`.svg-node/.svg-t/.svg-s`）适配双主题；**每张图必配 figcaption「图讲解」**，只画不讲=违规
- 表格优先（他 table-first 偏好）；原话引用用 `.quote` 块带说话人；推断打 `.infer` 徽章

## 全局揉法（第 6 节的做法）

结合他工作**必读真材料，不许凭记忆硬写**：

- 架构：`产品-端侧触发器/1-原始素材/{planner 架构, advisor 架构}`（HTML 用 python 抽文本读；`车载智能助手评测体系2.0.html` 101MB **别碰**）

- 他要"全局"= **真读全部**：派 3-4 个并行 general-purpose subagent 按时间分段读完整个纪要文件夹，每个只抽「模块在每场会往判断层爬还是往管道沉 + 定生死原话（谁:「」）+ 得/失 + 生态位读数 ↑↓↔」，主线程合成
- 第 6 节标配四件：①生态位起伏时间线 SVG（判断中枢↔管道折线，绿点=被抬/红点=被压）②**逐场详表**（日期·会 / 发生了什么含得失 / 定生死原话 / 读数色标）③收编者图谱表（每人立场+跨全程干了什么）④**真实场景逐步走查表**（每步：发生什么/价值归谁/他在干嘛，钉出哪一步是关口）——走查表是他最认的"直观"
- 原话用「」+ 说话人；凡推断标「推断」徽章让他核
