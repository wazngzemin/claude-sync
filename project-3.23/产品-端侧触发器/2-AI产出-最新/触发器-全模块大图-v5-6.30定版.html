2. 新增: 枚举/描述信号分类详解 — 两张对照表,枚举类(性别/位置/有无=标签→直接条件匹配)vs描述类(着装/毯子/项圈=非标签→动态SP),含丁彬+西钺确认原话+具体示例
3. 新增: 3种query类型对照 — 张兰若①条件触发②一次性③间接需求→对VLM任务相同,持续/一次区分在Advisor

5. 新增: VLM输出抖动/glitch风险框 — 张兰若警告"但凡出现一次错误·触发器就会有问题",需15-6抖动过滤配合
6. 新增: SP拼接谁做争论全过程 — 张兰若(VLM自己拼)→西钺(触发器拼)→丁彬(关键是谁清SP)→最终定触发器全管
7. 新增: 仁杰语义判断结论 — 端侧不做语义判断·做条件判断(运算符)·语义判断在云端
8. 新增: 舱外前置检查推演 — 西钺确认舱外也需触发器先判断现有信息是否足够
9. 新增: 静态/动态Advisor对照 — 静态4个永久(舒适/出行/情感/内容)vs动态用户触发创建用完销毁,含西钺原话
10. 新增: Context=固定拼接(不是订阅) — 西钺:"给advisor/planner作用一样,推理时固定拼接"+Advisor不做条件判断
11. 新增: VLM约束5条汇总 — 频率5秒/客观描述/KV槽位/可能不带标签/不做时间判断(张兰若)
12. 新增: AlwaysOn SP管理机制 — 基础+动态拼入/KV槽位/多任务合并/生命周期,丁彬+西钺原话

SP生命周期完整回路（6.30定版·最核心新增链路）：
① ⑰动态Advisor创建任务 → 给⑮人话需求("毯子掉没掉")
② ⑮触发器 → 15-16判断枚举/描述 → 描述类 → 15-17调提槽工具判direction
③ 舱内 → 15-18拼接SP(KV槽位+多任务合并) → 经L14+15-12下发㉔端侧触发器
④ ㉔端侧24-8'下发SP → VLM端模5秒周期推理 → 按KV槽位返回answer
⑤ 结果回写④信号库 → L9同步到⑫云端信号库 → ⑮ 15-13阈值穿越检测
⑥ 穿越命中 → 15-10打包事件 → L18 push给⑰动态Advisor
⑦ ⑰判断任务完成 → L34通知⑮清理SP → ⑮删除该SP → AlwaysOn还原
⑧ ⑰执行后续: 给⑲Director建议 → ⑳推理 → 回复用户 / 执行动作
丁彬原话walkthrough: "advisor负责开启+关闭任务 → 告诉触发器要感知什么 → 触发器拼SP → 下发VLM → KV值返回 → 穿越检测 → 传给advisor → advisor判退出 → 通知触发器停止 → 还原SP"
