# 价值捕获定位框架：判断你在产业链哪一层、会不会被挤掉、往哪条路拿回定价权

> 从"车企利润率 6.1%→3.2%，问题是生态位缺失"抽出的可复用操作模型。
> 原文讲的是车企；本页把它变成一套**通用透镜**——可用于诊断①你的产品（豆包座舱/Planner）②你的个人职业押注 ③反复出现的 badcase 根因。

## 核心问题

**你活得很累（卷、加班、猛产出），不等于你在捕获价值。** 利润 / 资源 / 影响力，都往两类生态位沉淀——"不可绕过的关键节点"和"别人做不出、决定能否上牌桌的能力"——而不往"活最重但可替代"的组装层沉淀。所以做决策前先问一个反直觉的问题：**我（或我的产品/模块）站在价值链的哪一层？这一层捕不捕获价值？如果不，我往哪条路迁移？**

---

## 场景映射

| 文章里的角色/机制 | 你的 Planner / 座舱 / 职业场景 | 可迁移动作 |
|---|---|---|
| 宁德时代 = 基础设施层（电池，正向飞轮） | 底层模型层（豆包 Seed 1.8 / 算力）= 座舱的"电池" | 判断这层对上汽是"不可绕过"还是"可平替配套"，飞轮转没转（*推断映射，非文章原文*） |
| 华为 = 能力供应商（智驾/座舱/OS，决定能否上牌桌） | **豆包座舱 / Planner = 座舱 AI 大脑，恰好这一层** | 用"没有豆包，D6X 上不上得了智能座舱牌桌"判断你这层值不值钱 |
| 车企 = 组装层（造一辆卖一辆，线性无复利） | 个人的一次性猛产出（写完一版 PRD 就结束，不沉淀资产） | 把线性产出改成复利资产（定义权/押注记录/影响力） |
| 比亚迪路线 = 制造驱动（成本领先） | 性能/成本工程：KV Cache 复用、上下文优化省 token | 在既有层把单位成本压到极致（稳，但不改生态位） |
| 华为/小鹏路线 = 技术驱动（智驾数据飞轮） | **badcase → SP 迭代 → Planner 变强的数据闭环** | 让每次 badcase 沉淀成越来越强的资产，而非一次性修补 |
| 出行服务商 = 服务驱动（用户关系+持续服务） | **触发器持续任务（S9-S10）+ 用户记忆锁定** = 座舱从工具变服务终端 | 把被动应答升级成主动持续服务（价值天花板 + 具身智能入口） |

> 映射诚信：座舱=能力层、底层模型=基础设施层是我把文章框架套到你产品上的**推断**，不是文章原文；真实的分成关系、上汽能否自研平替，需要你用内部信息判断。凡标 *推断* 处都请你亲自校验。

## 方法框架

### 四个诊断维度（判断任何一层值不值钱）

把文章里"宁德/华为/车企谁赚钱谁亏"背后的逻辑抽象成四个可打分的维度：

| 维度 | 一句话判据 | 高分长什么样 | 低分长什么样 |
|---|---|---|---|
| **① 不可绕过性**（choke-point，卡点） | 别人能绕过你吗？ | 关键节点，全链路被迫围着你配套（宁德） | 随时可被替代，绕过你没损失（普通组装） |
| **② 差异化能力**（capability moat，能力护城河） | 你提供的是别人做不出的能力，还是谁都能做的活？ | 决定终端"能否上牌桌"的入场券（华为智驾） | 谁都能做、只是你在做（总装） |
| **③ 复利/锁定**（compounding & lock-in，越滚越强+黏住） | 你每次交付是在积累越来越强的资产吗？ | 数据飞轮/用户关系/标准，越用越强（智驾闭环、出行服务） | 一次性交易，卖完断联，不沉淀（造一辆卖一辆） |
| **④ 价值层级**（value tier，你在阶梯的哪级） | 你在基础设施/能力/服务/组装的哪一层？ | 基础设施 · 能力 · 服务 | 纯组装 |

### 四层价值阶梯（价值从高到低不是绝对，看你在该层的不可替代性）

```
   ┌────────────────────────────────────────────┐
   │  服务层    占用户持续关系（出行服务商）      │ ← 最难最值钱，复利最强
   ├────────────────────────────────────────────┤
   │  基础设施层  卡不可绕过节点（宁德/电池）      │ ← 飞轮自增强
   ├────────────────────────────────────────────┤
   │  能力层    卖决定牌桌的差异化（华为智驾座舱）  │ ← 值钱，但易被下游反噬
   ├────────────────────────────────────────────┤
   │  组装层    线性交易、可替代（车企卖车）        │ ← 被两头挤，除非规模成本极致
   └────────────────────────────────────────────┘
```

**铁律**：不是"层级越高越安全"，是"你在该层的不可替代性有多强"。组装层做到比亚迪的成本极致也能活；能力层护城河浅也会被下游吃掉。**层级定天花板，不可替代性定生死。**

## 操作 SOP：生态位诊断五步

**输入**：一个待诊断对象（你的产品 / 你负责的模块 / 你个人职业 / 一个竞品）。
**输出**：它在哪一层、捕不捕获价值、护城河能撑多久、往哪条路迁移。

1. **画价值链**：把这个对象所在的完整链路列出来（上游→你→下游）。座舱例：算力/模型 → 座舱 AI（你）→ 整车厂 → 用户。
2. **四维打分**：对"你"这一层，四个维度各打 0-3 分（判据见上表）。总分 0-12。
3. **定位到层**：按最高分维度归层——不可绕过性高→基础设施；差异化能力高→能力层；复利锁定高→服务层；四维都低→组装层。
4. **压力测试护城河**：问三个掀桌问题——① 下游能自研平替你吗（能力层反噬）？② 上游/新技术能绕过你吗（基础设施被颠覆）？③ 你的复利是真在滚还是假象（每次交付有没有沉淀资产）？
5. **选迁移路径**：按你的筹码在三条路里选（见下方决策矩阵），列出前提条件清单。

**失败模式**：
- 把"我很忙很努力"当成"我在捕获价值"（努力 ≠ 生态位）。
- 只看当前分数、不做步骤 4 的护城河压力测试 → 读出虚假安全感。
- 把三条路当三选一 → 错过"叠加走"的最优解。

**怎么判断这次诊断有用**：诊断完你能说出一句"我这层最大的掀桌风险是 X，我下一步押 Y 路径、因为我有 Z 筹码"——能说出这句，诊断就成立；说不出，是停在了描述、没进到判断。

## 检查清单：表面竞争 vs 结构生态位（根因分诊器）

当利润/资源/影响力/评审话语权在下滑，先跑这个清单，别急着归因"竞争太激烈/我不够努力"：

- [ ] 同样的"激烈竞争"下，**有没有人在这条链上赚钱**？（有 → 病不在竞争，在你的层）
- [ ] 我这层的交付是**线性一次性**，还是**沉淀成越来越强的资产**？
- [ ] 别人**绕过我**有没有损失？（没损失 → 不可绕过性 = 0）
- [ ] 我提供的是**决定对方能否上牌桌的能力**，还是**锦上添花**？
- [ ] 我的**下游**有没有能力/动机**自研平替我**？（有 → 能力层正在被反噬）
- [ ] 我赚的是**一次性交易**，还是**持续关系/服务**？
- [ ] 我在为"把当前层做到极致"发力，还是在为"迁移到更高层"发力？（两者都行，但要知道自己在做哪个）

**≥3 项指向"结构"** → 这是生态位问题，加班/降价/再努力都治不了根，必须谈迁移。

## 价值捕获评分卡（可量化，配套 tool.html 的评分器）

对诊断对象四维各打 0-3：

| 分数 | 含义 |
|---|---|
| 0 | 完全没有（纯组装/可替代/一次性） |
| 1 | 微弱（有一点但不稳） |
| 2 | 明显（有真实优势） |
| 3 | 强（结构性、自增强） |

**总分解读**：
- **10-12**：强生态位（宁德/成熟服务商）。守住 + 加固飞轮。
- **7-9**：能力层，值钱但有反噬风险（华为智驾当前）。重点做步骤 4 的护城河压力测试。
- **4-6**：夹心层，往上够不着、往下被挤。必须选一条迁移路径，别原地卷。
- **0-3**：组装层。要么规模成本做到极致（比亚迪），要么尽快迁移，否则利润率注定 6.1%→3.2%。

## 三路径决策矩阵：怎么选迁移路径

| 路径 | 定价权来源 | 适合的筹码 | 前提条件（缺了就走不通） | 天花板 | 风险 |
|---|---|---|---|---|---|
| **制造驱动**（比亚迪） | 成本领先 | 规模、供应链整合、执行力 | 有规模经济、成本差能持续拉开 | 低（组装层里称王） | 对手规模追上/技术路线变，护城河被重挖 |
| **技术驱动**（华为/小鹏） | 差异化 + 数据飞轮 | 数据、软件迭代力、闭环设计 | 智驾成决定性差异 + 飞轮转得动 | 中高 | 同质化、数据边际递减、押注可能落空 |
| **服务驱动**（出行服务商） | 用户关系 + 持续服务 | 用户入口、运营、生态 | L4 自动驾驶落地 + 用户接受"买服务不买车" | 最高（终局赢家） | 依赖 L4，时间最不确定，最像愿景 |

**选择启发式**：
- 有规模没数据 → 先走制造，同时攒数据往技术切。
- 有数据闭环 → 押技术，这是复利最实的一条。
- 已占用户入口 → 往服务爬，别只当卖货的。
- **头部策略 = 叠加走**（特斯拉），不是三选一。

**座舱版落法（推断，需你校验）**：你现在同时手握技术驱动的筹码（badcase→SP 数据闭环）和服务驱动的引擎（触发器持续任务 + 用户记忆）。你产品的价值捕获升级 = 把这两条一起走：**用数据闭环加固"能力层不可替代性"（防上汽自研反噬）+ 用触发器/记忆往"服务层"爬（占用户持续关系）**。

## Prompt 模板：生态位分析器

复制到任何 LLM（大语言模型），填入 `{{对象}}`，产出结构化诊断：

```text
你是产业结构分析师。对下面这个对象做"价值捕获生态位"诊断，只讲结构、不讲努力。

对象：{{对象，如"豆包座舱在上汽荣威 D6X 的智能座舱产业链里"}}
所在价值链（上游→它→下游）：{{列出，如"算力/大模型 → 座舱 AI 大脑（它）→ 整车厂上汽 → 车主用户"}}

请按四维打分（0-3）并给判据：
1. 不可绕过性：别人绕过它有损失吗？
2. 差异化能力：它提供的是决定"能否上牌桌"的能力，还是锦上添花？
3. 复利/锁定：每次交付沉淀成越来越强的资产吗，还是一次性？
4. 价值层级：基础设施 / 能力 / 服务 / 组装，哪一层？

然后回答：
- 总分与所在层。
- 三个掀桌风险：下游能自研平替它吗？上游/新技术能绕过它吗？复利是真在滚还是假象？
- 迁移建议：按它的筹码，制造/技术/服务三条路里推荐哪条，缺哪个前提。
- 一句话结论：它最大的掀桌风险是 X，应押 Y 路径，因为有 Z 筹码。
```

这个 prompt 你能双用：既能诊断你的产品/职业，也能当作你 SP（系统提示词）里"结构化推理"的一个范式——**逼模型先归因到结构、再给动作**，正好对治你 badcase 里"信息全但推理错"那类（模型停在表层归因）。

---

## 60 分 vs 95 分（用这套框架时的质量分水岭）

| 维度 | 60 分（停在表层） | 95 分（真在用框架） |
|---|---|---|
| 归因 | "太卷了/我不够努力" | "同链上有人赚钱，说明是我这层不捕获价值" |
| 诊断 | 说自己"挺重要的" | 四维打分 + 一个掀桌风险数字 |
| 护城河 | 只看当前有没有优势 | 做压力测试："下游 18 个月内能自研平替我吗" |
| 路径 | "我要做智驾/做服务"（喊口号） | "押技术，因为我有数据闭环这个筹码，缺的前提是统一数据系统" |
| 落地 | 停在分析 | 把结论变成本周一个动作（改哪份 PRD/加哪个复利机制） |

## 判断标准

### 做对了的信号

- 你能对自己的产品说出一句："没有我这层，下游上不了 X 牌桌"——且这句经得起反驳。
- 你的每个"努力"都能回答"它在加固哪一维（不可绕过/差异化/复利）"，而不只是"我很忙"。
- 你识别出至少一个**生态位漂移**信号（下游正在自研、上游技术要换代）。
- 你的迁移选择带前提条件清单，不是喊口号。

### 仍然停留在表层的信号

- 把"竞争激烈/太卷/我不够努力"当终点归因。
- 诊断完只有形容词（"很重要""挺关键"），没有 0-3 打分和掀桌风险。
- 读出的是"我在好位置所以安全"（误读警告 4）。
- 三条路当三选一，或者选完不列前提。

## 三个应用场景（同一套框架，三处落点）

1. **产品**：诊断豆包座舱在上汽产业链的生态位——你是华为式能力层还是可平替配套？跑五步 SOP，输出"防反噬 + 往服务层爬"的动作。
2. **职业**（呼应你的核心短板：押注判断）：把"定义权 / 归属谈判 / 押注判断"看成从"组装工（执行型 PM，线性产出）"迁到"能力供应商（不可替代的判断力）"的三个杠杆。一次性猛产出 = 车企卖车模型；沉淀定义权/押注记录 = 往能力层爬。
3. **badcase 根因**：当某类坏例反复出现，用"表面竞争 vs 结构生态位"清单分诊——是战术问题（调 prompt/换工具描述），还是结构问题（这能力本就不该在这层做/这层没资源投入所以长期烂）。

## 需要你补充的材料（这些我不知道、不编）

- 豆包座舱与上汽的真实**分成/商业关系**：你们是"不可绕过的能力供应商"，还是"可被替换的供应商"？
- 上汽（或其它主机厂客户）**有没有能力/计划自研平替** Planner / 座舱 AI？（决定你能力层护城河深浅）
- 你的 badcase→SP 闭环**有没有真沉淀成复利资产**，还是每次一次性修补？
- 触发器持续任务的**商业化/服务化**路线图，到没到"占用户持续关系"的程度？

把这几条填进来，这套诊断就能从"通用框架"变成"你产品的真实作战地图"。
