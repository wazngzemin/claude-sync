7.8 评审复盘 · 行动手册

不再重复诊断（见v3），这一版只回答：问题在哪、为什么、怎么练、练到什么程度算过关

第一章 · 这次最大的四个问题

按致命程度排序。每个问题拆三层：发生了什么 → 它为什么致命 → 它是怎么形成的。

问题A：定位被打了6分钟，你一个字没说

发生了什么

12:26 汪帅问"能不能默认打开"。12:32 丁彬替你挡了一句"为了统一控制逻辑"。15:03 汪帅追了一句"airbox可没人告诉我们信号"。16:04 汪帅说"不能为了有触发器而强加功能"。

从12:26到18:18，你沉默了5分52秒。中间张荣16:52点名叫你，你又花了1分26秒才开口，开口说的是一个事实确认问题。

这6分钟里，触发器的定位从"统一控制入口"被打成了"不必要的中间层"。你全程旁观。

它为什么致命

这6分钟是整场会的胜负手。在这个窗口里：

  • 只有汪帅一个人在攻击，还没有同盟
  • 张荣还在中立位，没有倾向
  • 丁彬刚说了"统一控制逻辑"，你可以接过来展开
  • 蒋文17:18主动提了"进停车场关舱外"这个场景，天然弹药

换句话说，12:26到18:18是你的阵地战窗口。窗口关闭之后（38:07 汪帅给出完整替代方案、41:27 Walter附议），就变成了一个人对一群人的结构性劣势。你在窗口期沉默，在劣势期更沉默。

它是怎么形成的

不是因为你怂。是因为你分不清"技术问题"和"定位问题"。

汪帅说的每一句话都包着技术外衣——"能不能默认打开""不需要外界触发""强加功能"。你在技术层面判断：他说的对，默认开确实可行。所以你无话可说。

但他打的不是技术。他打的是"触发器这个模块有没有必要存在"。这个问题的答案不在"三个always-on怎么做"里面，在"0818动态任务调度入口放在哪"里面。你没有切到这个层面去回应，因为你脑子里没有"定位层"这个抽屉。你只有"技术层"一个抽屉，打开一看，对方说得对，就关上了。

问题B：你帮对方缩小了你自己的scope

发生了什么

18:37 你说"那可能就只能判断一个Always on的button了"。22:40 你说"触发器可能它主要现在，在目前这个阶段，可能它主要是两块事情"。33:30 你说"说白了就两块"。

三句话，三次自缩。你在告诉全场：触发器就做两件小事。

它为什么致命

你自己把触发器标了个低价。汪帅47:35说"我没觉得它过触发器能有十足的理由来说服我自己"——他的底气从哪来的？从你自己的话里来的。你说"就两块"，他说"两块不需要独立模块"，逻辑无懈可击。你递了刀。

它是怎么形成的

你潜意识里觉得把scope说小等于降低被攻击面——"我就做这么点小事，你总不会为了这么点事攻击我吧？"。这个逻辑在考试里成立（题目简单=容易得分），在谈判里完全相反。你把scope说得越小，对方就越容易论证"这么小的事不需要独立模块"。

另一层原因：你确实没想清楚触发器0818的scope是什么。你的所有准备都围绕0804的三个always-on做，没有把0818的动态任务调度、条件触发、状态回调放进你的叙事里。你不是不敢说大，是你的脑子里确实只有这"两块"。

问题C：裸体撤退，第三次

发生了什么

48:06 张荣说"泽民，我们就把触发器的P0需求给他删了吧"。你说"OK"。然后立刻跳到"那下边但是在订阅这一块..."。

没有任何反对。没有任何条件。没有锁任何东西进纪要。直接翻页。

它为什么致命

张荣说的是"删了吧"——征求意见。你有一个窗口说"删P0可以，但两件事写进纪要"。你没用这个窗口。结果：

  • 0818触发器scope没有保护——"0804删了"会被默认理解为"触发器这条线砍了"
  • button状态链路的产品owner变成了无主——汪帅说"不一定过触发器"，你没说"那产品口径我来跟"
  • 你在纪要里的角色 = "被删需求的人"。你本可以是"主动砍掉0804、锁定0818的人"

它是怎么形成的

三场评审都一样：被否的瞬间你急于翻页。这不是勇不勇的问题（违禁词不用），这是一个条件反射——"当众被否"触发了"赶紧离开这个话题"的逃避行为。7.2是"OK行吧"，7.4是沉默接受，7.8是"OK那下边"。

你把"被否"和"丢脸"绑在了一起。但在评审桌上，被否是常态。被否之后说了什么才决定你是输还是平。你一次都没有在被否之后说过第二句话。

问题D：没有带目标进会议室

发生了什么

你走进7.8这场会，脑子里的目标是"把触发器的方案讲清楚"。而不是"拿到一个决策走出去"。

区别是什么？"讲清楚"的终点是别人点头说"我听懂了"。"拿到决策"的终点是"纪要里多了一行，署你的名"。你的行为模式——展示方案、等人提问、回答问题、被否就走——完全是"讲清楚"模式。你从来没有主动说过"这个点今天能定吗"。

它为什么致命

不带目标进会=评审完了什么都没变。你花了55分钟，走出会议室，你的scope比走进去之前还小了。如果你带了目标——比如"今天定一个结论：0818触发器做不做"——最坏的结果也是"定了不做"写进纪要。"定了不做"比"没讨论过"好一百倍，因为下次你可以重新提。而"没讨论过"意味着大家默认它不存在。

它是怎么形成的

你对评审会的定义是"方案展示"——我带着方案来，大家提意见，我改完走人。这是执行型PM的评审观。高级PM的评审观是"资源谈判"——我带着三个要拿的决策来，拿到几个走人。

你从来没有在会前写过"这场会我要拿走什么"。这不是你不会，是你没有这个习惯。这个习惯的缺失是因为你一直在做的是"被安排评审"而不是"主动发起评审"。被安排的人来讲方案，主动发起的人来拿决策。

第二章 · 四个问题背后的两个根因

四个问题不是四个独立的毛病，它们长在同一棵树上。

根因1：你把评审当考试，不是当谈判

考试的规则是：我准备充分 → 展示 → 评委打分 → 通过或不通过。在这个框架里，"准备充分"=写好PRD/讲解稿，"展示"=讲方案，"打分"=别人提问你回答，"不通过"=回去改。

你的所有行为都符合这个框架。你花大量时间写文档（备考），上会讲方案（交卷），被否就走（没考好下次再来）。你没有"出价""还价""锁条件""带走战利品"这些动作，因为考试里不需要这些。

但评审不是考试，是谈判。谈判的规则是：我带着要拿的东西来 → 出价 → 博弈 → 拿到一些东西走。在这个框架里，"被否"不是失败，是博弈的一个环节。被否之后说什么才是胜负手。

你要切换的不是话术，是操作系统。

根因2：你的准备全投在方案上，零投在博弈上

你在7.2到7.8之间做了什么？写PRD、改讲解稿、做复盘HTML、准备技术方案。你在准备"方案"——技术上怎么做、逻辑怎么讲。

你没准备的是：汪帅会怎么打你？上次他用"打定位"赢了，这次他会不会再来？如果他再来我怎么接？张荣是什么立场？我能不能会前跟她对齐？这场会我要拿走什么？

你90%的准备时间在打磨方案，0%在准备博弈。然后你带着一个完美的方案走进一场博弈，被不打方案只打定位的人赢了。

你缺的不是方案能力，缺的是博弈准备的习惯。

两个根因的关系：根因1（评审=考试）导致了根因2（准备全投方案）。因为在考试框架里，"准备"就等于"准备方案"。只有切换到谈判框架，"准备博弈"才变成一个有意义的动作。

所以你要改的第一件事不是某个具体话术，是对"评审"这件事的理解方式。

第三章 · 需要强化的四个能力

从四个问题和两个根因推出来的，不是泛泛的"提升沟通能力"。每个能力有具体的定义、判断标准、练习方式。

# 能力 定义 你现在的水平 过关线
A 攻击面识别 听到一句话能判断它是技术问题、成本问题、归属问题还是定位问题，并选择对应的回应层 只有"技术"一个分类，其他全归为"我不知道怎么接"然后沉默 5秒内能给对方的话打标签（技术/成本/归属/定位），选对层再开口
B 定位防御 当有人质疑你的模块/需求/项目为什么存在时，能用一句话说清存在理由，不被拖入技术细节层 零。三场评审从未回应过定位攻击 能在被问"为什么需要X"的10秒内说出"X存在因为..."的一句话定位
C 撤退锁定 方案被否/需求被砍时，在接受的同时锁定1-2个条件写进纪要 三次裸体撤退，每次只说"OK" 每次被否后能说出"我接受，但有一个条件写进纪要"并实际拿到
D 目标设定 会前明确"这场会我要拿走什么"，会中追着目标走，会后检查拿到没有 从未带目标进过评审 每场评审前能写出1-3个"要拿走的决策"，会后检查命中率

四个能力的优先级：B定位防御 > A攻击面识别 > D目标设定 > C撤退锁定。理由：B是你反复被打的同一个洞（汪帅两次用同一招赢你），堵这个洞的投入产出比最高。A是B的前置条件（你得先识别出"这是定位攻击"才能启动定位防御）。D是长期习惯改变。C是D的子集——有了目标，撤退时自然知道该锁什么。

第四章 · 怎么练（具体到每天做什么）

练法A：攻击面识别 — 每天15分钟录像回放

做什么

每天花15分钟，回看你当天参加的任何会议（不限于评审）。不看内容，只标注每个人每句发言的"层"：

层关键词信号正确回应层
技术层"怎么实现""能不能做""方案是什么"给技术答案
成本层"凭什么""谁背工期""排期满了""做不了"谈对价，不给技术答案
归属层"这个挂谁""谁负责""归哪个模块"先索对价再接/拒
定位层"为什么需要X""这东西有必要吗""可以不用X"一句话定位+架构价值，绝不进细节

每句话标完之后，看你实际怎么回应的——层对了没有？如果你用技术层回应了一句定位层的话，标红。

为什么这么练

你的技术耳不是不会判断，是没有建立分类习惯。现在你的脑子里只有一个桶（"这个问题我知不知道答案"），需要建四个桶。15分钟/天，连续两周，四个桶就会变成自动反应。

什么时候算过关

连续三场会议，你标注的时候发现自己在会上每次都选对了层（技术问题给技术答案、成本问题谈对价），标红次数降到零。

练法B：定位防御 — 写一页纸，背到脱口而出

做什么

现在就写，写完贴在你的笔记本第一页。内容只有三样：

触发器一句话定位（≤30字）：

你自己想这句话。不要用AI。想不出来可以从这个方向想：触发器解决的问题是"谁来统一管理所有触发逻辑"——如果没有触发器，每个场景（开机开舱外、进停车场关舱外、button状态变更、动态任务切换）各自做各自的，到了0818就没有统一入口了。

参考骨架（你来填肉）："触发器是_____的统一入口，没有它_____就会_____"。

触发器的三个版本scope（三句话）：

0804做什么 → 0818做什么 → 0904做什么。上会只讲当前版本scope + 一句话带后续版本价值。绝不只说当前版本。

汪帅的三种攻击 + 预置台词（三个Q&A对）：

汪帅会问你说
"为什么需要触发器？" （你自己写的一句话定位，脱口而出）+"如果不要触发器，0818动态任务切换谁调度？帅哥你有替代方案我听一下。"
"这个功能不需要过触发器" "如果只看这一个功能，你说得对，不需要。但触发器管的不是一个功能，是所有触发逻辑的统一入口。砍触发器不是砍一个功能，是砍一整条链路。"
"这东西可以更轻量" "轻量我同意，0804我已经把scope砍到最小了。但轻量不等于不需要独立模块——路由器也很轻量，但你不会把路由功能分散到每个设备里。"

为什么这么练

定位防御不是临场发挥，是肌肉记忆。你在12:26沉默了6分钟不是因为你不聪明，是因为你没有现成的台词可以调用。脑子里有现成台词的人，被问到的第一秒就能脱口而出。脑子里没有的人，需要现场组织语言，然后在组织的过程中被恐惧打断，然后沉默。

写下来、背下来、对着镜子说三遍。你需要的是从"我要想一下怎么说"变成"我张嘴就说了"。这中间的距离不是智商，是练习次数。

什么时候算过关

有人问"为什么需要触发器"，你不需要想就能说出那句话。从被问到开口≤3秒。

练法C：撤退锁定 — 背三件套，条件反射级

做什么

遇到"方案被否/需求被砍/scope被缩"的瞬间，强制执行三件套，说完三句才允许翻页：

① 止损："这个方案/需求我撤。"
（你在主动撤，不是被动删。同一个事实，两种定价）

② 保判断："但我判断0818仍然需要____。"
（撤方案不撤判断。方案被否≠判断被否）

③ 锁条件："今天进纪要一件事：____。荣姐这条OK吗？"
（撤退带走一样东西。判断权在最后一句话里易手——你的最后一句话不能是"OK"，要是一个问句，让对方回答你）

怎么练

接下来两周，不限于评审，任何场合只要你的提议被否了（哪怕是"中午吃什么"这种小事），强制执行三件套。目标是把三件套变成条件反射——被否的第一秒自动启动，不需要想。

在小事上练到自动化，到了评审桌上你才不会忘记。

什么时候算过关

下一场评审，如果有提议被否，你说完三件套之后再翻页。不管三件套说得好不好，说了就算过关。第一次不需要说得漂亮，需要说了。

练法D：目标设定 — 会前5分钟一张A4纸

做什么

每场评审开始前5分钟，手写一张A4纸（不是打字，手写），内容三行：

① 这场会我要拿走的决策是什么？（1-3个，写具体。"对齐方案"不算决策，"XX写进纪要"算决策）

② 谁会反对？他会说什么？我怎么接？（写最可能的1-2个攻击）

③ 如果全部被否，我的撤退锁定条件是什么？（至少一个"但是"）

这张纸放在你面前，会中每10分钟扫一眼。会后把纸翻过来，写一行：今天拿到了几个。

为什么手写

手写慢，逼你只写最重要的。打字快，你会写出一整页"背景/目的/预期"——那是PRD，不是作战纸条。你需要的是3行纸条，不是一页文档。

什么时候算过关

连续5场评审都带着这张纸进去、写了回顾。不要求命中率——第一阶段只要求"有目标"就行。命中率是第二阶段的事。

第六章 · 下一场评审的完整备战清单

假设下一场评审是0818 scope的讨论。以下是你走进会议室之前必须完成的清单。

  • 写完定位一页纸（练法B），确保一句话定位能脱口而出

会前1天

  • 写A4纸条（练法D），三行：要拿走的决策 / 谁会反对怎么接 / 撤退锁定条件
  • 对着镜子模拟一遍：汪帅问"0818也不需要触发器"，你怎么接（≤3秒开口）
  • 对着镜子模拟一遍：方案被否，你执行三件套

会前5分钟

  • 把A4纸条放在面前。扫一眼：我今天要拿走什么
  • 深呼吸。提醒自己：这是谈判，不是考试。我来拿东西的，不是来交卷的

会中

  • 有人质疑定位 → 启动定位防御（一句话定位+架构价值），不进技术细节
  • 有人说"做不了/没必要" → 先判断是技术层还是成本层。成本层用对价回应，不辩技术
  • 方案被否 → 执行撤退三件套，说完三句才翻页
  • 每10分钟扫一眼A4纸条：我要拿的决策拿到了没有？

会后5分钟

  • 翻过A4纸条，写一行：今天拿到了几个决策。交出了几次判断权。有没有执行三件套
  • 如果三件套没用到（没被否），算好事。如果被否了但忘了用三件套，标红，下次重点练

第七章 · 怎么判断自己在进步

三个跟踪指标，每场评审后打分：

指标 现在（7.8基线） 60分线 80分线
拿到决策数 0 / 0 / 0（三场全零） 每场≥1个写进纪要 每场≥2个，且是你主动推进的而非被动确认的
交出判断权次数 3次（7.8） 每场≤1次 0次
定位攻击响应时间 6分钟（12:26→18:18） ≤30秒 ≤5秒，且回应在定位层不在技术层

进步不是线性的。预期路径：

第1-2场：你会记得A4纸条，但会中紧张时忘掉。这正常。会后回顾的时候发现"我又忘了"，这个发现本身就是进步——你以前连"忘了"都不知道

第3-4场：你会在某个瞬间识别出"这是定位攻击"，但来不及说出台词。来不及没关系——你识别出来了，这比7.8进了一大步

第5-6场：你会说出台词，但可能说得不够好，或者时机晚了几秒。不要追求完美——说了比没说强一百倍

第7-8场：你会开始觉得"带目标进会议"和"被否后锁条件"是理所当然的事。到了这一步，你已经不再是同一种PM了

第八章 · 你会遇到的三个心理卡点

提前说出来，到时候遇到你知道这不是"又犯错了"，这是练习过程中的正常环节。

卡点1："我说了定位台词但对方反驳了，更尴尬了"

会发生。你说"触发器是统一调度入口"，汪帅说"你所谓的统一入口目前只有三个开关"。然后呢？

然后你已经赢了一步。因为讨论从"触发器要不要存在"变成了"触发器目前scope够不够大"——这是一个你可以用数据回答的问题（0818还有哪些场景），不是一个你无法反驳的定位否定。你把战场从定位层拉回了技术层，这就是进步。

不要追求"我说了一句话对方就接受了"。这个预期不现实，也不是目标。目标是"讨论在我设定的框架里进行"。

卡点2："我知道该锁条件但嘴上说不出来，太突兀了"

头几次确实会觉得突兀。你以前从来没在被否之后说过"但我有一个条件"——第一次说的时候你会觉得全场都在看你，觉得自己在无理取闹。

你不是在无理取闹。你在做评审中最正常的事：提条件。全场每个人都在提条件，汪帅在提（"我可以不过触发器"=他在提"我的工期不加"的条件），刘杨在提（"排期满了"=他在提"你要做自己排"的条件），只有你不提。你开始提了，他们不会觉得你突兀，会觉得你终于正常了。

卡点3："我练了两周感觉没用，是不是这方法不适合我"

两周不够。你的"考试型评审"习惯用了多少年形成的？不可能两周改掉。真正的判断节点是第6-8场评审。如果到了第8场你还是跟7.8一模一样——定位被打6分钟沉默、裸体撤退、零决策——那才是方法有问题。在那之前，每一场有一个改善（哪怕只是"我这次3分钟就开口了"），就是在正确的路上。

拿走的东西

四个问题：①定位被打不回应 ②自缩scope ③裸体撤退 ④没带目标进会

两个根因：①把评审当考试不当谈判 ②准备全投方案零投博弈

四个能力要练：A攻击面识别 → B定位防御 → C撤退锁定 → D目标设定

四周每周加一个，不要一次全上

下场评审之前三件事：①找张荣拿一句话 ②找仁杰过一遍方案 ③对着镜子模拟汪帅攻击

判断进步的三个数字：拿到决策数 / 交出判断权次数 / 定位攻击响应时间
