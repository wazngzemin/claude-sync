AI 场景编排 · 需求全景图

从"按钮触发预制场景"到"语音驱动智能编排"的产品升级
负责人：王泽民 · 更新日期：2026-06-04 · 版本：v1.0

一、这个需求是什么

4月16日，Daniel 提出：用 Planner（Director）调用原子化车控能力，替代传统预制场景的固定执行逻辑，实现"用户说人话 → 豆包理解意图 → 智能编排车控 → 自动执行"的体验升级。

❌ 旧模式（预制场景）

  • 入口：情景空间 APP / 场景大师 APP / 桌面卡片
  • 触发：用户手动点按钮 / 说"打开XX模式"
  • 执行：写死的规则化逻辑，每个模式 10-20 条固定执行项
  • 退出：手动点退出 / 自动退出条件触发
  • 回滚：退出后所有设置项自动恢复原状
  • 持续监控：运行期间持续监测异常（高压/SOC/档位），自动退出+通知
  • 优点：确定性 100%，安全有兜底
  • 缺点：不灵活，不能理解自然语言，不能结合环境个性化编排

✅ 新模式（AI 场景编排）

  • 入口：唯一入口 = 语音（APP 页面仅做引导）
  • 触发：用户自然语言表达（"我想休息一下"/"车里有只猫帮我看着"）
  • 执行：Planner 理解意图 → 匹配场景知识 → 动态编排车控组合
  • 退出：语音退出（无自动退出监控机制）
  • 回滚：无回滚，仅恢复到座椅记忆位置
  • 持续监控：暂无（需 Adviser + 触发器配合）
  • 优点：理解自然语言，结合环境个性化，泛化 query 覆盖
  • 缺点：不能 100% 保证执行正确，无回滚，无持续监控
一句话定义：AI 场景编排 = 把上汽原来"按钮 → 固定规则执行"的预制场景，升级为"语音 → Planner 理解意图 → 动态知识检索 → 智能编排车控"的 AI 原生方式。用户不再需要知道"宠物模式"这个名字，只需要说"我要下车，车里有只狗"，Planner 就能理解并执行。

三、场景分类与处理路径

上汽原有约 25 个预制场景，按 Planner 可替代程度分为三类，走不同的技术路径。

分类决策树

分类原则：执行项是否100%确定？→ 是：走预置（分子级调用）→ 否：是否需要结合环境灵活编排？→ 是：走 Planner 编排 → 否：暂不支持

🅰 分子级调用（预置场景，不走 Planner）— 6 个

特征：执行项 100% 确定、有安全边界约束、有完整回滚、Planner 做不到 100% 保证。需提需求给方向靠（左晨）注册语音指令。

🐾 宠物模式
13 条执行项 · P档+电量≥20% · 空调22°C+外循环+屏蔽按键+大屏"主人马上回来" · 完整回滚 · 持续监控SOC/高压/档位
🚿 洗车模式
12+ 条执行项 · P档+电量≥18%+未插枪 · 关窗/锁门/禁用电动门/内循环 · 完整回滚 · 速度/电量/4h超时自动退出
🏕 露营模式
多条执行项 · 二次确认 · 持续监控高压/SOC/档位 · 车门未关超10分钟报警 · APP+短信通知
🔋 离车保电
车窗/高压联动 · 设置时间到期/SOC≤15%/非P档自动退出 · 完整回滚
🛏 大床模式
座椅全放倒 · 确定性执行
💺 一键躺平
座椅零重力 · 确定性执行

🅱 Planner 编排（动态知识注入）— 8 个（合并后）

特征：每次用户需求有差异，需要结合环境/偏好灵活编排。通过 Skill 知识告诉 Planner "这个场景大概要调哪些车控"，Planner 结合端状态和用户意图做推理。

四、核心业务流程

从用户说一句话到车控执行完成的端到端流程，以及两条不同路径的分流逻辑。

4.1 完整业务流程

① 用户语音输入

两种 Query 形式均需支持：

  • 泛化型："我想休息一下" / "车里有只猫帮我看着" / "好热好臭赶紧开窗"
  • 直接型："打开宠物模式" / "帮我开启洗车模式"
↓ ASR 语音转文字 → 拒识仲裁

② Selector SP 场景分类（v11）

4 步判定逻辑：ID_000 预判过滤 → 上下文辅助 → 场景特殊规则 → 互斥锚定

  • 输出：场景 ID（ID_001 ~ ID_013）或 ID_000（不走场景）
  • 支持多场景命中：如"好热好臭" → ID_004（降温）+ ID_012（通风）
  • 问题 当前命中率仅 60-70%，1/3 的 query 被错分或漏分
↓ 分流：分子级（直接执行） vs 编排级（走 Planner）

③-A 分子级路径（6 个预置场景）

  • Selector 命中预置场景 ID
  • 不注入 Skill 给 Planner
  • 直接调用方向靠注册的预置应用指令
  • 预置应用内部执行固定逻辑
  • 有完整的前置条件校验、回滚、持续监控

卡点 方向靠尚未注册这些指令，需提 query 示例给左晨

③-B 编排级路径（8 个编排场景）

  • Selector 命中编排场景 ID
  • 检索对应的 Skill 知识 + 动态示例
  • 连同端状态 env_info + 用户记忆 memory
  • 一起注入 Planner SP
  • Planner 推理输出 action_list

风险 动态知识注入会增加 SP token，影响耗时和注意力

↓

④-A 预置应用执行

  • 二次确认弹窗（安全场景）
  • 固定执行项逐条执行
  • TTS 播报
  • 中控屏显示专属页面
  • AIBOT 表情切换
  • 持续监控（高压/SOC/档位）
  • 异常时自动退出 + APP/短信通知

④-B Planner 编排执行

  • Planner 输出 action_list（多个工具调用）
  • 通过 SOA 下发车控指令给方向靠
  • TTS 播报 talk_content
  • AI 画布展示执行项
  • AIBOT 表情切换（1分钟后自动恢复）
  • 无回滚 退出仅恢复座椅记忆位置
  • 无持续监控 需要 Adviser + 触发器配合
↓

⑤ 用户退出场景

分子级退出：

  • 主动：语音/屏幕按键
  • 自动：车速/SOC/高压/超时/档位变化
  • 退出后：所有设置项 自动回滚
  • AIBOT 表情：预置应用通知 Director 结束

编排级退出：

  • 仅支持：语音说"退出"
  • 无自动退出机制
  • 退出后：不回滚，仅恢复座椅记忆位置（依赖记忆工具）
  • AIBOT 表情：触发后 1 分钟自动恢复默认（临时方案）
