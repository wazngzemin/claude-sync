# 3.30 Director 模块核心待办进展对齐会

> 来源：会议纪要-0330.md（38KB）
> 时间：2026-03-30 10:00-10:47

## 参会人
张坤（主持人）、郝晓伟（产品）、李明骏（开发）、李强（算法）、罗丽（测试）、李晨延、王泽民、刘正伟、赵磊

## 四大议题

### 1. 评测集构建
- **晓伟**整理了约 **150 条 query**，按模块分类覆盖全场景
- 分类：对话、车控（明确+模糊）、人脸注册/记忆、POI/导航、内容媒体、信息查询、主动服务、跟进任务、高阶体验
- **关键问题**：context 影响因素（端状态、舱外视觉等）未补充，需要按影响因素拆分 query
- **数据格式**：晓伟+李强负责定义，需支持多轮评测
- **分工模式**：7人分工，每人负责一个大模块构建 context
- **目标**：4月底 **2000 条**高质量数据，5 月启动训练

### 2. SP 优化（P1 长度优化）
- **李强**已完成修改（在 text SP 基础上改）
- 需要明骏现场 review
- 依赖正伟的工程支持才能上线实测
- 评测集出来后用评测集验证效果

### 3. 上游链路优化
- Context 模块问题：
  - 筛选准确率不够
  - 记忆时空连续性问题
  - 事件信息缺少
  - 工具使用情况缺类型
  - 瞬时记忆使用不好
- 交给**仁杰 context 专项**处理
- 本周排优先级，然后定开发排期

### 4. GUI Agent Bug 分析
- **泽民**分析了 51 个 GUI 相关 bug（总共 200+ planner bug）
- 需要按工具分类：是否工具问题 → 哪个工具 → 什么问题
- 不能靠二手信息，需要算法+产品一起 by 工具逐个分析
- GUI Agent 当前用途：点外卖、订座、订咖啡、交停车费

## 关键决策
1. 以晓伟的 150 条 query 为种子数据基础
2. 评测优先（PE 着急用），训练数据后续补
3. 比例：20% 评测 + 100% 训练（即 1000 条训练配 200 条评测）
4. 工具分析要 by 工具逐个过，不靠二手提炼
