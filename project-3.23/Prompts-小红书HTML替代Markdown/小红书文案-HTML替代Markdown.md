### 正文

你有没有发现，让 AI 写 Markdown 文档，一到 100 行以上就完全不想读了？😩

Anthropic Claude Code 团队的工程师 Thariq 最近发了一篇爆文，说他几乎已经完全不用 Markdown 了——全换成 HTML。我读完直接打开新世界，分享给大家👇

---

**🔑 为什么是 HTML？**

**1️⃣ 信息密度碾压 Markdown**
HTML 能表达的东西太多了：
✅ 表格、图表、流程图（SVG）
✅ 带高亮的代码片段
✅ 交互控件（滑块、按钮、拖拽）
✅ 甚至能嵌入 JS 做双向互动

Markdown 做不了的，HTML 全都能。

**2️⃣ 长文档终于能读完了**
超过 100 行的 Markdown，眼睛直接花掉。HTML 可以分标签页、加侧边栏、做响应式布局，手机电脑都能舒服读。

**3️⃣ 分享太方便**
Markdown 还得当附件发。HTML 直接丢 S3 生成链接，同事点开就能看，阅读率暴增。

**4️⃣ 文档可以交互！**
这是最颠覆的一点——HTML 文件里能加滑块、按钮、拖拽卡片，你调完参数一键「复制为 prompt」粘回 Claude Code，太丝滑了。

**5️⃣ 上下文摄取能力强**
Claude Code 能读你的代码库、git 历史、Slack 消息、Linear ticket……然后综合生成一份巨详细的 HTML 报告。这比手动整理效率高 10 倍。

---

**🎯 六大使用场景（附神级 prompt）**

**场景 1：方案探索**
> 「我不确定这个页面该走哪个方向。生成 6 种明显不同的方案，排成 HTML 网格，标注每个方案的取舍。」

**场景 2：代码审查**
> 「帮我审查这个 PR，渲染实际 diff，边栏加注释，按严重程度标颜色。」

**场景 3：设计原型**
> 「原型化一个新按钮的点击动画，加几个滑块让我调参数，再加一个复制按钮导出配置。」

**场景 4：技术报告**
> 「读取 rate limiter 相关代码，生成单页 HTML：流程图 + 关键代码片段 + 注释 + gotchas。」

**场景 5：自定义编辑器**
> 「给这 30 个需求做拖拽看板（Now/Next/Later/Cut），加导出按钮。」

**场景 6：学习理解**
> 「我不懂这个模块怎么工作，给我做一个交互式解释器。」

---

**❓ 你可能担心的问题**

🔸 **吃 token？** —— 确实比 MD 多 2-4 倍，但 Opus 4.7 有 100 万上下文窗口，完全够用。
🔸 **版本控制？** —— 这是 HTML 最大缺点，diff 很难看。
🔸 **做得丑？** —— 让 Claude 先读你的代码库生成设计系统文件，之后所有 HTML 都参考它。
🔸 **怎么查看？** —— 本地浏览器直接打开就行。

---

**💬 我的感受**

作者说了句很打动我的话：

> 「使用 HTML 之后，我反而比以往任何时候都更深入地在与 Claude 协作的回路中。」

不是 AI 替你做决定，而是你们在同一个丰富的画布上协作。这个心态转变真的太重要了。

---

**🧠 总结一句**

Markdown 是给人类写的，HTML 是人类 + AI 一起写的。下次让 Claude 输出的时候，试试说「做一个 HTML artifact」，你会被惊艳到的。
