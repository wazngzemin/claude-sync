1背景与目标

1.1 背景

动态示例是修复 Planner Badcase 的核心手段：一条「用户 Query → 决策分析」的示例对，在推理时按相似度检索、注入到 SP 的 {{scenario_sample}} 槽位，引导模型对同类 Query 做出正确决策。

目前动态示例以 Excel 手工维护，存在以下问题：

当前痛点说明
无校验工具名写错、Query 空、决策分析格式不统一，靠人工肉眼检查，历史表里已出现工具名混乱、张冠李戴。
易冲突/重复相似 Query 被重复录入、且决策不一致，污染检索结果，无查重手段。
交付链路长产品改 Excel → 人工同步进系统，中间环节多、易出错、无法追溯谁改了什么。
无环境隔离不同模型环境（SFT 版本）需要不同示例集，Excel 无法区分。

1.2 目标

  • 提供一个 Web 配置后台，CQC 同学可直接完成动态示例的查找 / 新增 / 修改 / 删除，替代 Excel。
  • 入库前做基础校验，堵住工具名写错、Query 查重等低级问题。
  • 支持按环境配置，先落地 SFT 版本，后续可扩展多环境。V2
  • 形态对齐现有干预工具，降低配置复杂度。
一句话定位
把动态示例从「Excel 手工台账」升级为「带校验、可追溯、CQC 自助 CRUD 的配置后台」，形态类比干预工具。

2名词解释

名词说明
动态示例「Query → 决策分析」示例对，注入 SP {{scenario_sample}} 槽位以修复 Badcase。
决策分析针对该类 Query，Planner 应如何决策的一句话说明（调用哪个工具、如何改写 Query、结合哪些状态）。
检索索引动态示例按 Query 建立的检索索引；推理时用当前 Query 命中 Top-K 示例注入。
环境一套模型运行环境（如不同 SFT 版本 / 测试 / 线上），不同环境可挂不同示例集。
CQC本后台的主要使用方，负责逐条录入与维护动态示例。

3需求范围

3.1 V1 范围（SFT 版本）本期

功能说明
列表 / 查找展示全部动态示例，支持按 Query 关键词搜索、按状态筛选、分页。
新增表单录入一条动态示例，提交前校验 + 相似示例提示。
修改编辑已有示例并保存。
删除删除单条示例（二次确认）。
基础校验非空、长度、工具名合法性、Query 查重。
操作日志记录增删改操作人与时间，可追溯。建议

3.2 V2 范围（后续）下期

  • 环境配置：每条示例可归属到指定环境，支持环境切换、跨环境复制。
  • 检索测试：输入一个测试 Query，预览会命中哪些示例，便于 CQC 验证示例是否生效。建议

3.3 不做（Out of Scope）

不做项原因
批量导入 / 导出会议明确：CQC 逐条录入即可，不做批量。（存量迁移见 §9，为一次性开发动作，非用户功能）
审批流会议明确：不走评审，保存即入库。
示例效果自动评测不在本工具范围，由测试链路负责。

4功能需求

4.1 列表与查找 V1

  • 默认以表格展示全部动态示例，列：Query、决策分析（超长截断，悬浮/展开看全文）、状态、创建人、更新时间、操作。
  • 顶部搜索框：按 Query 关键词模糊搜索；建议同时支持相似度检索（输入一句话，找语义相近的已有示例）。
  • 筛选：按状态（生效 / 停用）、按环境V2。
  • 排序：按更新时间倒序默认；支持点列头排序。
  • 分页：每页 20/50/100 可选。

4.2 新增 V1

点「新增」→ 填写表单→ 相似示例提示→ 提交校验→ 入库 + 建索引
  • 表单字段见 §5 字段定义。
  • 相似示例提示建议：填完 Query 后，实时检索已有相似示例并展示，避免录入冲突/重复条目。若存在高度相似示例，给出提醒但不强制阻止。
  • 提交时执行 §4.5 校验规则；通过后写库并更新检索索引。

4.3 修改 V1

  • 列表点「编辑」，表单预填当前值。
  • 保存时同样走校验。
  • 若 Query 发生变化，需重建该条检索索引（见 §7）。

4.4 删除 V1

  • 列表点「删除」，二次确认弹窗后删除，并从检索索引移除。
  • 建议采用软删除（标记删除、保留记录），避免误删不可恢复；或提供「停用」状态作为删除前的缓冲。

4.5 校验规则 V1

校验项规则不通过
Query 非空不可为空、去除首尾空格后长度 ≥ 1阻止提交
决策分析非空不可为空阻止提交
长度限制Query ≤ 200 字；决策分析 ≤ 1000 字（阈值可调）阻止提交
工具名合法性决策分析中提及的工具名，须在合法工具清单内（vehicle_basic_control 等 26 个）。检测到不在清单内的疑似工具名 → 提示。警告
Query 查重完全重复 → 阻止；高度相似 → 警告并展示相似条目阻止/警告
缺陷链接格式若填写，校验为合法 Meego 链接格式警告
工具名校验的价值
历史 Excel 出现过工具名写错、录音类工具张冠李戴。将合法工具清单内置为校验字典，可在录入环节直接拦截，是本后台相对 Excel 最大的质量增量。

4.6 环境配置 V2

  • 顶部提供环境切换器，切换后列表/编辑均作用于该环境。
  • 每条示例归属一个或多个环境；支持将示例复制到其他环境。
  • 先落地单一 SFT 环境（V1），V2 再放开多环境。

4.7 操作日志 建议

  • 记录每次新增/修改/删除的：操作人、时间、动作类型、变更前后内容。
  • 因动态示例直接影响线上 Planner 决策，保留审计可追溯问题来源。

5字段定义

字段类型必填说明
ID系统自动唯一标识，系统生成
Query文本是触发该示例的用户 Query
决策分析长文本是Planner 应如何决策的一句话说明
其他说明长文本否备注、补充上下文
缺陷链接链接否对应 Meego 缺陷单链接
Logid文本否关联的 trace_id / log_id，便于回溯原始 case
状态枚举是生效 / 停用（默认生效）
环境枚举是V2所属环境；V1 默认单一 SFT 环境
创建人 / 创建时间系统自动录入者与时间
更新人 / 更新时间系统自动最后修改者与时间

字段对齐现有 Excel 结构（Query / 决策分析 / 其他说明 / 缺陷链接 / Logid），并补充状态、环境、审计字段。

7生效机制（开发向）

动态示例按 Query 检索注入，因此增删改需同步维护检索索引：

操作索引处理
新增对 Query 建立检索向量/索引项，加入索引
修改 Query重建该条索引
仅改决策分析无需重建索引，更新内容即可
删除 / 停用从索引移除（停用=不参与检索）
待与开发确认：同步时机
推荐保存即近实时同步到检索索引（秒级）。若架构上索引重建成本高，可退化为定时同步（如每 N 分钟）或手动「发布」按钮触发。此项影响 CQC 改完多久生效，需 文涛 评估。
生效范围
「状态=停用」的示例不参与检索注入，作为删除前的安全缓冲；「生效」即进入 Top-K 候选。是否即时影响线上、还是随下次发版生效，需明确。

8权限与操作日志

角色权限
CQC增 / 删 / 改 / 查（主要使用方）
产品（晓伟/泽民）查看 + 审计日志；可编辑
其他按需只读

建议保留操作日志：操作人 / 时间 / 动作 / 变更前后内容，支持按人和时间检索。动态示例影响线上决策，出问题时可快速定位是哪条、谁改的。

9存量数据迁移

  • 现有 Excel 动态示例（约 517 条）需一次性导入后台，作为初始数据。
  • 此为开发侧一次性脚本动作，非面向用户的批量导入功能。
  • 迁移映射：Excel 的 Query / 决策分析 / 其他说明 / 缺陷链接 / Logid → 对应字段；状态默认「生效」。
  • 迁移后建议跑一次工具名校验，输出异常清单供人工修正。

11验收标准

  • CQC 可在后台完成一条动态示例的新增、检索、修改、删除全流程，无需接触 Excel。
  • 录入空 Query / 空决策分析被拦截；写错工具名有提示；重复 Query 有查重提醒。
  • 新增/修改后，检索链路能按约定时机命中该示例（配合 §7 生效机制验证）。
  • 存量 517 条完成迁移，列表可正常检索展示。
  • 每次增删改在操作日志中可查。
