行车场景

支持在智驾行车模式下，用户可像对一个真人司机一样，对智驾发起自然语言指令，例如：激活智驾、靠边临停、路口直行/转向/掉头等。

4.1 行车导航类_无参照物临停-630


用户下发没有任何参照物的靠边临停指令，无需输入目的地，仅需就近 / 按参照物停靠，如「前面靠边停」。智驾系统执行靠边临停动作，整个靠边动作、寻找安全停靠点、安全兜底由智驾保证。

4.4 行车导航类-VLM视觉参照物临停

4.4.1 整体描述

在智驾行车模式下，用户通过语音描述视觉可见的参照物发起靠边临停。系统通过 VL能力实时识别，将用户的自然语言描述与视觉场景进行语义对齐，定位目标参照物后执行靠边停车。
与POI类临停的核心区别：不依赖地图POI数据，通过实时 VL能力完成目标识别与定位。
典型场景：
场景01-用户即时任务：e.g. 「停在拉行李箱的人旁边」
场景02-用户长时任务：e.g. 「智驾去幼儿园接小宝，找个阴凉的地方停靠」


4.4.2 场景01-用户即时任务

指在智驾行车过程中发起即时、临时性的，以视觉目标物参照的靠边停车任务；
ODD：即时性任务可用车速范围 ≤ 30kph，加上车速抖动，实际支持35kph（TBD）。即：
当车速≤35kph时，才支持视觉类VLM参照物临停；
当车速＞35kph时，提示用户车速有点快，看不到｛参照物｝。

4.4.2 场景02-用户长时任务

指用户发起的并非即时性的任务，而是有一定计划性、长时的任务，例如，在上车后提前规划并query「智驾去幼儿园接小宝，到时候找个阴凉的地方停靠」。参照物「阴凉的地方」在发起query时不在视野内，系统需要在导航行驶过程中挂起临停子任务，待车辆到达目的地附近后，开始通过VLM寻找参照物，智驾执行靠边停车。
