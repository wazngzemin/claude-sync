创建日期：2026-03-27 | 版本：v1.0
背景
当前系统核心问题是模型注意力分散：PE 变长导致不稳定、推理不结合 context、幻觉增加。影响范围涵盖 GUI Agent、Planner 工具、用户个性化需求等。
三大优化方案
方案一：示例浓缩提纯
保留 10-15 个最核心示例，覆盖四大能力维度
能力 1：多工具协同使用
能力 2：串行与并行判断
能力 3：困难场景灵活处理
能力 4：个性化需求理解
实施：核心示例筛选 → 示例优化 → 动态知识库构建 → 前置筛选层设计
方案二：工具复杂度简化（架构重构）
核心理念：职责分离
Planner 改为只输出自然语言规划
引入 Flash 模型做参数转换 + 知识注入
架构：Director → Planner(自然语言) → Flash模型(参数转换) → 工具调用
方案三：补充知识和示例
Badcase 驱动的优化：收集分类 → 根因分析 → 判断缺知识/缺示例 → 针对性补充
动态检索：任务分类 → 知识匹配 → 相似度排序 → 动态注入
GUI Agent 专项（紧急，截止 3-30）
