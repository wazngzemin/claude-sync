## 概述

本文档定义了TOB线简单复杂仲裁模块针对AI汽车升级的长期方案，将query按仲裁模块和意图模块分类处理，覆盖条件任务、持续任务、Face ID CRUD、记忆CRUD、记忆推理、情景+状态推理、联网推理等场景，并包含大量评测数据对比。

## 关键章节

### 1. 长期方案
- **仲裁模块负责**：判断query是否简单（可直接执行）或复杂（需进Planner）
- **意图模块负责**：判断query的具体意图类型
- 两者组合决策最终路由

### 2. 覆盖场景

| 场景 | 处理方式 | 说明 |
|------|----------|------|
| 条件任务 | 仲裁判断为复杂 | 需进Planner处理 |
| 持续任务 | 仲裁判断为复杂 | 需进Planner处理 |
| Face ID CRUD | 仲裁+意图组合判断 | 涉及人脸数据操作 |
| 记忆CRUD | 仲裁+意图组合判断 | 涉及记忆增删改查 |
| 记忆推理 | 仲裁判断为复杂 | 需结合记忆推理 |
| 情景+状态推理 | 仲裁判断为复杂 | 需结合车内外情景 |
| 联网推理 | 仲裁判断为复杂 | 需联网获取信息 |

### 3. 评测数据
- 包含大量TOB模型结果与预期结果的对比表格
- 覆盖多种query类型和场景
- 用于评估仲裁模型的准确率

## 核心要求

- 仲裁标准不仅是句式简单复杂，而是句式+意图
- SFT能处理的高精召走，其他都进Planner
- 后续只用仲裁模型判断是否给Planner
- 目标ACC>=96%，P50<=150ms，P90<=250ms

## 提取的实体

- **TOB**：面向企业的技术方案线
- **仲裁模块**：判断query简单/复杂的核心模块
- **意图模块**：判断query具体意图类型的模块
- **Face ID CRUD**：人脸识别数据的增删改查操作
- **记忆CRUD**：用户记忆数据的增删改查操作

## 提取的概念

- **简单复杂仲裁**：将用户query分类为简单（直接执行）或复杂（需规划）的机制
- **高精召**：高精度召回，指模型有把握正确处理的query
- **组合判断**：仲裁模块和意图模块联合决策的路由策略
