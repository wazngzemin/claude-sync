触发器（豆包上车 Driver Agent 2.0）架构定论，读完全部 PRD+6场会议+Advisor/Director架构后得出：

**本质**：触发器不是独立产品，是**动态 Advisor（及 Planner/主动服务）感知订阅的执行底座**。动态 Advisor 把 Director 派的目标拆成 4 类任务，其中条件/定时/VLM 三类向触发器注册（感知订阅）。没有触发器，动态 Advisor 二期(DNA-616)的条件/定时/POI/VLM 全落不了地。

**两个触发器分工**：
- 云端触发器 = 注册校验+中转（Advisor/Planner 动态任务唯一入口）、规则/场景全集配置、跨任务语义仲裁、模型类事件、场景&任务下发。上汽是纯云端形态、直接承载判定。
- 端侧触发器 = 两级条件判定、信号扫描+500ms快照对齐、P0-P4+互斥组、冷却/抢占恢复、端侧定时器[需7]、VLM调度+五态结果、结果回写信号库→上报Context、弱网动作闭环。赛力斯是端云结合、有端侧VLM。

**3 条修正铁律（评审必守，撞 5/7 西钺结论的硬伤）**：
1. 动态注册必须经云端触发器中转，不直连端侧；
2. 端侧结果统一走 信号库→Context，端侧不直推 Advisor/Director（notify 在云端感知订阅层做）；
3. 信号判定只在一侧（不双判）——上汽走云、赛力斯走端。

**VLM 触发真身**：阈值穿越（last=False & this=True 上升沿才触发，防轰炸），event/state 两模式。上汽当期无端侧视觉、VLM 主战场是赛力斯。

**大图节点实名（业务流程-全模块大图-v2，2026-06-21 核对）**：㉔=端侧触发器(端侧·VLM任务调度器) / ㉕=**感知器(端侧)**（干"VLM定时捞图→VQA判事件"盯宝宝/周边/风景，**别叫"端侧VLM"**，那是泽民一度的口误）/ ④=端侧信号库(=端侧Context) / ⑮=云端触发器 / ㉑=云端Context / ㉒=工具库(非"工具执行") / ⑨拒识 ⑩意图解析 ⑬火山后台 ⑯静态Advisor ⑰动态Advisor ⑲Director汇入 ⑳Director推理 ㉖TTS ㉘车控执行。关键连线：L14 ⑮→㉔下发、L16 ㉔→④回写、L22 ⑰→㉕感知订阅下发、L23 ㉕→⑰、L33 ⑳→⑮注册。VQA链路：⑰→视频通话接口→端侧摄像头→云端VQA模型→回⑰+注入㉑；⑳经㉒search_visual_info调同一VQA。**圈号①-㉚是架构节点编号，写流程图时别拿来当步骤序号（会撞节点）。**

**感知订阅不是独立模块（用户 2026-06-22 强订正，曾因此被骂"瞎搞"）**：感知订阅 ⊂ 广义触发器。广义触发器（=617"触发与感知"模块，泽民 owns）一个 scope 内含：触发/条件半(㉔/⑮/④) + 感知半(㉕感知器) + 感知订阅(对动态Advisor暴露的"订阅"注册接口)。⑰↔触发器 7 接口=订阅(=感知订阅)/回调/事件/能力描述/文本→事件类型/优先级/兜底。**别把"感知订阅 / 感知器㉕ / 触发器"拆成三个并列模块**——它们是同一个广义触发器的不同部分。"文本→事件类型"=注册时 NL→事件类型的接口(第5个)，不是大图节点、不是㉕。

**6.22 需求评审结论（重大，改了 scope，逐字稿在 `产品-端侧触发器/6.22会议.md`，复盘在 `6.22评审复盘.html`）**：① **端侧触发器 scope 大幅收窄**——固定/常驻/标配的端侧闭环类(哨兵/上下车/起步安全/迎宾/简单车控)全部回**赛力斯 CDC 底软**，不走触发器；端侧触发器只做 { 本地定时器 + 动态视觉感知事件 + 通用视觉事件通道 + 任务中心端侧同步 }。② **链路修正(佳杰当场抓)**：事件由 ⑮云端触发器组装后 **push 给 Advisor(15→17)**，Context 只存 condition/状态(车里有小孩)，Advisor 订阅事件不轮询 Context。③ **施佳杰=佳杰**(同一人=动态Advisor+事件模块+任务中心+AIBox调度)。④ always-on/VLM 任务调度在**佳杰 AIBox**，不在触发器。⑤ **3 卡点 7 月中定**：SP 下发走哪条 / 端模能否做"看到X就告诉我"条件式注意力(**没试过**) / 事件组装放云端or端侧。⑥ 新增依赖：通用视觉事件通道(分舱内静态/动态/舱外，prompt增量更新)、任务中心端云打通(任务vs场景字段+enable/disable写入接口)。⑦ 泽民评审表现：准备的7点清单不错但临场偏被动、链路图画错被当场纠、末尾跟丢——下次要主动收敛+界定自己scope。⚠ PRD-v2/五方关系文档 over-claim 了端侧 scope，待按 6.22 重划。
