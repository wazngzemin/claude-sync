MODULE 1AGI三幕路线图
学习目标：理解AGI到来的路径，判断你的产品/项目处于哪一幕。

核心判断

李广密提出AGI的实现不是一次性跳跃，而是三个连续阶段：

与 Hassabis 判断的张力

李广密：2年内，路径是连续的，不需要新范式

Hassabis：50%概率需要1-2个架构级突破

综合判断：方向正确但可能需要"架构级补丁"——2-5年区间，取决于突破是否出现

对你的产品意味着什么

问自己：我的产品在第几幕？

  • 第一幕产品（纯对话）：高风险，窗口极短，需快速收割或转型
  • 第二幕产品（能调用工具执行）：中等风险，需建数据飞轮
  • 涉及物理世界：低风险，AGI到来反而加速你

Checkpoint 1

能回答：(1) 三幕之间的递进关系是什么？(2) 你的产品在第几幕？(3) 为什么第二幕→第三幕的跳跃可能不是连续的？

MODULE 2Coding是环境
学习目标：理解"Coding ≠ 编程能力，= AI操作世界的接口"这一认知升级。

概念重定义

传统理解：Coding = 一种技能（像数学、写作）。
李广密的重定义：Coding = 一个环境（像物理世界是人类的环境）。

强化学习框架类比

在强化学习中：Agent在环境中行动 → 获得反馈 → 学习。

Coding就是AI的"环境"——AI通过生成和执行代码来：

  • 采集外部信息（API调用、爬虫、传感器读取）
  • 处理数据（算法、转换、分析）
  • 执行行动（发请求、控制设备、写文件）
  • 获得反馈（执行结果、错误信息、状态变化）

对 Planner 的直接启示

Planner 本质上也是一个"准Coding Agent"——它通过调用23个工具（= 执行操作）来改变车内世界。进化方向：从"选择固定工具"到"组合/生成新操作序列"。

Checkpoint 2

能解释：为什么"Coding能力领先 = AGI竞赛领先"？因为Coding是AI操作世界的唯一通道——没有执行力，理解力再强也只能"说"不能"做"。

MODULE 3Agent持续学习缺口
学习目标：理解为什么当前所有Agent产品都需要人类持续维护，以及根本原因。

Hassabis的诊断

Agent做不到"交付后不管"的根本原因：不能在具体使用环境中持续学习和适应。

为什么Memory ≠ 持续学习

机制能做的不能做的
Memory检索回忆"用户上次说过X"学会"这类用户通常想要Y"
Goal List追踪"当前目标是Z"学会"在这种情境下应优先Z"
SP示例固定的69个参考case从新badcase自动生成新规则

对Planner团队的直接影响：当前SP是固定的，模型不因使用变好。Badcase需要人工排查→手动修SP。这正是Hassabis指出的Agent根本缺陷。

Checkpoint 3

能区分"有记忆"和"持续学习"的本质差异。能解释为什么所有Agent产品都需要人类持续维护。

MODULE 4AlphaFold三条件筛选器
学习目标：掌握一个判断"这个问题AI能不能解决"的实用框架。

三个条件

Hassabis从AlphaFold的成功中提炼出AI突破的三个必要条件：

如何使用这个筛选器

对你想用AI解决的任何问题，逐一检查三条件：

条件满足情况判断行动
三条件都满足AI几乎一定能突破投入资源，时间问题
缺目标函数需要人定义"好"先花时间设计评估标准
缺数据/模拟器AI暂时不能做先建数据管道或模拟环境
搜索空间不大可能不需要AI传统方法可能就够

用三条件评估Driver Agent Planner

① 搜索空间：大（23工具 × 多步规划 × 参数组合）✓

② 目标函数：模糊（"好的驾驶体验"难以量化）← 这是瓶颈

③ 数据：有（用户交互log + tool_feedback）✓

→ 突破方向：定义更清晰的"好规划"评估标准（延迟? 完成率? 用户满意度?）

Checkpoint 4

能用三条件评估一个具体问题。能识别Planner的瓶颈在"目标函数模糊"。

MODULE 5壁垒窗口与竞争格局
学习目标：理解AI产品壁垒的时间窗口，判断什么才是真正的护城河。

0.5-1年追赶定律

Hassabis指出：前沿大模型的能力，0.5-1年后会出现在端侧小模型上。这意味着：

竞争格局"三幕定位"

玩家三幕定位判断
OpenAI第一幕成功(ChatGPT)，第二幕分散战略分散+组织动荡，可能错失Coding
Google/DeepMind全幕跟随，科学应用独特优势"最领先的追随者"，天花板低但不会掉队
Anthropic第二幕聚焦(Claude Code)隐含被看好（李广密未明说但逻辑指向）
Meta开源路线，加速小模型追赶压缩壁垒窗口的关键力量

危险信号检查：如果你的产品壁垒只是"我们用了更好的模型/更好的prompt"——这个壁垒的半衰期只有6个月。

Checkpoint 5

能区分真假壁垒。能回答"你的产品去掉AI还能用吗？"这个试金石问题。Driver Agent的真壁垒是什么？（车端数据 + 荣威合作关系 + 物理世界部署）
